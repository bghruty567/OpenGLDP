# TestGradient 使用说明（Facade版）

## 1. 目标
TestGradient 作为测试程序，只调用 Facade 接口完成：
- 数据加载
- 数组选择
- OpenGL 梯度计算
- VTK 梯度计算
- 时间与误差对比打印

不在测试程序中堆叠核心实现逻辑。

## 2. 命令行参数
程序参数格式：

OpenGLDP.exe <vtk文件路径> <point|cell> <数组名> <重复次数>

- vtk文件路径：必选（不传则使用默认路径）
- point|cell：数组关联类型，默认 point
- 数组名：可选，不传则自动选该关联类型下第一个
- 重复次数：可选，默认 5，用于统计平均时间与最小时间

示例：
- OpenGLDP.exe Data\uGridEx.vtk point scalars 5
- OpenGLDP.exe Data\uGridEx.vtk point vectors 10

## 3. 输出解释
### 3.1 数组列表
程序会先打印当前数据集在指定关联类型下的可选数组：
- 数组名
- 分量数
- tuple 数量

### 3.2 时间对比
- VTK_time_ms_avg/min：VTK 梯度计算平均/最小时间
- GL_wall_ms_avg/min：OpenGL端到端平均/最小时间（含调度与读回）
- GL_gpu_ms_avg/min：OpenGL 纯GPU计算平均/最小时间（GL_TIME_ELAPSED）

### 3.3 精度对比
- MAE：平均绝对误差
- RMSE：均方根误差
- MAXE：最大绝对误差
- 另外会打印前若干条 GL 与 VTK 的结果向量逐项对照

## 4. 与 Qt 界面对接方式
TestGradient 的流程与未来 Qt 交互一一对应：

1) 读取文件按钮  
-> CAEProcessingFacade::loadDatasetFromVTKFile

2) 左侧数组列表  
-> CAEProcessingFacade::listFields

3) 点击计算  
-> CAEProcessingFacade::computeGradient

4) 结果列表展示  
-> CAEProcessingFacade::listDatasets / 结果元数据

5) 结果渲染  
-> CAEProcessingFacade::exportDatasetToVTK + 选择结果数组

## 5. 当前约定
- Auto 方法：
  - 规则网格：FD
  - 非结构网格：WLS
- 点数据与单元数据都可通过 request.association 选择

## 6. 常见问题
1) 只打印了“Gradient xxx -> xxx”没有时间对比  
原因：旧版本 TestGradient 在前半段提前 return。  
解决：使用当前 Facade 完整测试流程版本。

2) 指定数组后计算失败  
检查：
- 数组名拼写是否与打印列表一致
- 关联类型 point/cell 是否匹配
- 该数组是否存在且分量数 > 0
