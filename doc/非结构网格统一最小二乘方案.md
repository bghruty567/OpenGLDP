# 非结构网格统一最小二乘方案

## 1. 这次收敛后的最终方案

当前非结构网格梯度模块的最终公开方案是：

- 对点数据和单元数据统一采用局部支撑上的加权最小二乘梯度重构；
- 默认模式为 `AWLS`；
- `WLS.glsl` 只保留为内部回退内核；
- GUI 不再公开旧的多分支算法模式。

## 2. 为什么要把旧方案收敛掉

旧版本的问题不在于“完全不能算”，而在于：

1. 分支太多，不利于论文说明。
2. 有些分支混入了明显的 VTK 求导逻辑，不利于表述成统一的 OpenGL 方案。
3. 不同网格类型分别走不同线，维护和实验都很乱。

因此这次收敛的目标是：

- 保留最小二乘思想；
- 去掉过多按单元类型拆开的默认算法线；
- 让点数据和单元数据都尽量服从同一套数据流。

## 3. 当前主链

当前主链是：

```text
computeGradient(...)
-> computeByAdaptiveWLS(...)
-> ensureAdaptiveSupport(...)
-> computeUnstructuredAdaptiveWLS(...)
-> AdaptiveWLS.glsl
```

失败时内部回退：

```text
computeUnstructuredWLS(...)
-> WLS.glsl
```

## 4. 统一框架里的“统一”到底体现在哪

统一体现在四件事上：

1. 样本统一
   - 点数据用点坐标
   - 单元数据用单元中心

2. 邻接统一
   - 点数据用点邻接
   - 单元数据用单元邻接

3. 支撑描述统一
   - `offsets`
   - `neighbors`
   - `frames`
   - `dimTags`
   - `quality`
   - `meanNeighborDistance`

4. GPU 求解统一
   - 最后都落到 WLS/AWLS 形式的局部最小二乘求解

## 5. 支撑怎么构造

### 5.1 基础邻域

先从原始邻接图中拿一环邻域。

### 5.2 扩邻域

如果邻域数量不足，且开启了 `useAdaptiveNeighborhood`：

- 先扩二环
- 再用 KNN 补充

### 5.3 局部谱分析

在候选邻域上计算局部协方差矩阵并做特征分解，用来得到：

- 主方向 `frame`
- 维数标签 `dimTag`
- 邻域质量 `quality`

### 5.4 平衡选邻居

在局部框架下按方向分桶，尽量减少“邻居全在一边”的情况。

这一步本质上是在改善最小二乘矩阵条件。

## 6. 点数据和单元数据的差异

虽然主框架统一，但两者仍有一个工程上的差别：

- 单元数据默认支撑更紧一些

当前默认参数下，代码会对单元数据使用更小的默认 stencil：

- `minNeighbors = 4`
- `targetNeighbors = 8`
- `maxNeighbors = 12`
- `radiusScale = 1.75`

这样做的原因是：壳状网格或薄层网格上的 cell-centered 场对过大的邻域更敏感。

## 7. 这是不是完全照搬某一篇论文

不是逐字照搬，但路线是明确的文献同源：

- 核心求解思想来自非结构网格加权最小二乘梯度重构；
- 局部支撑、权重、正规方程的做法属于标准 WLS 家族；
- 自适应扩邻域、维数判别、质量驱动正则增强，属于面向工程稳定性的实现增强。

因此更准确的说法是：

> 当前方法是“以文献中的 WLS 梯度重构为理论基础、结合本系统数据结构和 GPU 执行模型做出的工程化统一实现”。

## 8. 这一版能不能作为毕设最终方案

如果你的当前实验结论确实是：

- 解析场 exact gradient 验证全部 `NMAE_vec < 1`、`NRMSE_vec < 1`
- 真实数据里只有 `ShipHull_0`、`notch_stress` 这类困难样例相对 VTK 仍在 `1` 到 `2` 之间

那么这版方案可以作为本科毕设的最终工程方案。

但论文里应当这样写：

1. 解析场验证是主要正确性依据。
2. 真实数据与 VTK 的对照是工程比较，不是真值比较。
3. 困难样例偏差需要写成局限，而不是回避。

## 9. 论文中推荐使用的表述

建议使用下面这类措辞：

> 针对非结构网格，本文不再按单元类型切换多套默认求导路径，而是采用统一的局部支撑加权最小二乘梯度重构框架。系统首先依据点邻接或单元邻接构造局部支撑，并通过局部谱分析获得方向框架、维数标签和邻域质量指标，随后在 GPU 上执行自适应加权最小二乘求解。该方案在解析场 exact gradient 验证中表现稳定，并作为本文的最终工程实现。
