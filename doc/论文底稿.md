## 第一章 绪论

### 1.1 研究背景

在 CAE 后处理、科学可视化和工程数据分析中，梯度场是极其基础的派生量。无论是应力集中区识别、流场边界层观察、结构特征提取，还是后续的噪声抑制与多尺度分析，都需要稳定且可重复的梯度计算结果。传统工作流通常依赖 CPU 侧有限元/可视化工具进行处理，但在大规模网格与多字段批量实验场景下，其交互性与吞吐能力受到限制。

与规则网格相比，非结构化网格在工程中更常见，但其梯度重构更复杂。一方面，单元拓扑多样、局部几何尺度变化大；另一方面，点数据与单元数据之间存在不同的离散语义，导致算法设计、结果解释与参考基线选择都需要更谨慎。在此背景下，构建一套面向规则网格与非结构网格的统一 GPU 数值处理原型系统，具有明确的研究意义和工程价值。

### 1.2 问题提出

本课题主要面向以下问题：

1. 如何将规则网格和非结构化网格统一映射到一套可供 GPU 高效处理的内部数据表示。
2. 如何在非结构化网格上实现基于形函数导数的梯度计算，并与现有工程基线建立可比实验。
3. 如何把梯度计算与数据优化模块纳入统一测试框架，形成可复现实验链路。

### 1.3 研究目标

本文的目标不是构建一个完全取代 VTK 的全栈系统，而是实现一条清晰、可验证、可扩展的数值处理主路径，其核心目标包括：

1. 实现规则网格的 GPU 有限差分梯度计算。
2. 实现非结构化网格的 GPU 形函数导数梯度计算。
3. 实现图双边滤波驱动的多尺度分解与融合。
4. 建立 GUI、基准测试程序、CSV 报告与 ParaView 导出构成的实验闭环。

### 1.4 本文主要工作

结合当前工程实现，本文的主要工作可概括为：

4. 实现基于图双边滤波与多尺度融合的数据优化引擎 [GLFilterEngine.h](../GLFilterEngine.h)。
5. 构建梯度测试程序 [TestGradient.cpp](../TestGradient.cpp)、多尺度测试程序 [TestMultiScale.cpp](../TestMultiScale.cpp) 与字段评估程序 [TestFieldMetrics.cpp](../TestFieldMetrics.cpp)。

### 2.2 规则网格梯度理论基础

规则网格梯度计算采用有限差分思想。对于标量场 `u`，其在物理空间中的梯度定义为：

`grad(u) = [du/dx, du/dy, du/dz]^T`

在规则或曲线规则网格上，可先在参数空间上对场值做一维差分，再通过局部 Jacobian 映射到物理空间。当前实现位于 [Shaders/FD.glsl](../Shaders/FD.glsl)，其特点是：

1. 内部点采用中心差分。
2. 边界点退化为单边差分。
3. 通过 Jacobian 逆映射把参数空间导数转换为物理空间梯度。

### 2.3 非结构化网格形函数导数法

对有限元单元，几何坐标和场变量都可通过形函数插值表示。设单元有 `n` 个节点，形函数为 `N_a`，则：

`x(xi) = sum(N_a(xi) * x_a), a=1..n`

`u(xi) = sum(N_a(xi) * u_a), a=1..n`

对参数坐标求导可得：

`du/dxi = sum((dN_a/dxi) * u_a)`

设 Jacobian 为：

`J = dx/dxi`

则物理空间梯度满足：

`grad_x(u) = J^{-T} * grad_xi(u)`  （满维单元）

对于嵌入三维空间的二维曲面单元或一维线单元，更稳健的表达是采用度量张量：

`G = J^T * J`

`grad_x(u) = J * G^{-1} * grad_xi(u)`

这也是当前形函数导数法着色器的核心思想。实现位置见：

- [Shaders/ShapePointGradient.glsl](../Shaders/ShapePointGradient.glsl)
- [Shaders/ShapeCellGradient.glsl](../Shaders/ShapeCellGradient.glsl)
- [Shaders/CellDataToPointLift.glsl](../Shaders/CellDataToPointLift.glsl)

### 2.4 加权最小二乘与其关系

加权最小二乘法是非结构化网格梯度重构中常见的另一条技术路线，其基本思想是在局部邻域内构造超定线性系统，通过最小二乘求解梯度。当前工程中保留了 AWLS 路径，接口定义见 [CAEInterfaceTypes.h](../CAEInterfaceTypes.h)，实现入口见 [GLGradientEngine.h](../GLGradientEngine.h)。

不过，本文后续实验主线不再以 AWLS 为主要验证对象，而是以形函数导数法为核心。这一调整的原因在于：形函数导数法与有限元单元插值理论更直接一致，更适合作为当前毕设中“算法原理明确、可追溯到单元数学定义”的主线方法。

### 2.5 图双边滤波与多尺度融合

双边滤波同时考虑几何邻近性与数值相似性，能够在保边的同时抑制局部高频扰动。将其推广到点邻域图或单元邻域图后，可写为：

`u_i' = sum_j w_s(i,j) * w_r(i,j) * u_j / sum_j w_s(i,j) * w_r(i,j)`

其中：

- `w_s` 表示几何邻近权重；
- `w_r` 表示数值相似权重。

在多尺度处理中，可通过重复平滑获得不同尺度的平滑层，再将相邻平滑层做差得到细节层，最后按设定增益重建输出场。系统对应实现见：

- [GLFilterEngine.h](../GLFilterEngine.h)
- [GLFilterEngine.cpp](../GLFilterEngine.cpp)
- [Shaders/Bilateral.glsl](../Shaders/Bilateral.glsl)
- [Shaders/MultiScaleFuse.glsl](../Shaders/MultiScaleFuse.glsl)

### 2.6 相关文献基础

从相关文献看，梯度计算方法的测试通常不是单一维度，而是至少分为“代码/算法正确性验证”“网格与几何敏感性分析”“工程基线对照”三层。

1. 对有限差分或一般 PDE 数值代码，主流做法是采用制造解或解析解进行代码验证，并通过网格加密观察误差收敛阶。这类方法的核心目标不是和某个软件输出接近，而是验证数值实现是否满足理论阶次与离散方程要求。典型参考包括 Salari 与 Knupp 关于 MMS 的综述，以及 Roy 等将 MMS 用于结构/非结构 CFD 代码验证的工作。
2. 对非结构网格梯度重构方法，文献通常会进一步区分单元类型、网格质量和几何形态。例如 Wang 等在各向同性非结构网格上结合制造解与网格收敛研究梯度重构误差，Diskin 与 Thomas 则专门考察高长宽比、随机扰动和曲率对梯度精度的影响。这说明非结构梯度方法的实验不应只做单一数据集平均误差，而应关注点/单元关联、曲面/体网格和畸变敏感性。
3. 对有限元或固体力学背景下的导数、应力和应变恢复，文献普遍把主变量与导数量分开评价。Aycock 等在固体力学 MMS 验证中明确报告位移与应力/应变的不同收敛行为，说明导数类量本身就比原始场更敏感，因此点数据与单元数据梯度路径分开讨论是合理的。
4. 对工程软件对照实验，文献和工程实践通常把其定位为“一致性验证”而非“真值验证”。因此，本文后续实验中将“解析场对真值”与“真实字段对 VTK”分开组织，避免把软件间接近误写为算法绝对正确。
5. 对 VTK 参考实现本身，其官方文档表明 `vtkGradientFilter` 提供了 `ContributingCellOption`、`ReplacementValueOption` 和 `FasterApproximation` 等选项，用于在混合维度数据、缺少可贡献单元以及速度/精度权衡场景下进行保护或退化处理；并行方面则通过 `vtkSMPTools` 选择 `Sequential`、`STDThread`、`TBB` 或 `OpenMP` 后端并设置线程数。因此，与 VTK 比较时必须同时报告其算法配置和并行配置。

## 第三章 系统设计

### 3.1 总体架构

系统总体上采用“GUI/测试程序 + 门面层 + 内部数据层 + GPU 计算层 + VTK 桥接层”的分层结构。门面层统一协调数据加载、数值计算与结果导出，使 GUI 与测试程序共享同一套核心流程。

核心文件如下：

- 门面层：[CAEProcessingFacade.h](../CAEProcessingFacade.h)
- 内部数据层：[DataObject.h](../DataObject.h)
- VTK 桥接层：[VTKDataConverter.h](../VTKDataConverter.h)
- GPU 运行时：[OpenGLManager.h](../OpenGLManager.h)
- 梯度引擎：[GLGradientEngine.h](../GLGradientEngine.h)
- 数据优化引擎：[GLFilterEngine.h](../GLFilterEngine.h)

### 3.2 内部数据表示

为避免 GPU 计算直接依赖 VTK 内部存储，本系统定义了统一的数据对象 `DataObject`。该结构包含：

1. 点坐标数组 `points`
2. 单元中心数组 `cellCenters`
3. 点/单元字段数组 `dataArrays`
4. 点邻域、单元邻域的 CSR 结构
5. 单元连接关系、偏移和类型编码
6. 规则网格维度 `dimensions`

这一设计使规则网格与非结构化网格可以共享下游测试、统计和导出逻辑。

### 3.3 门面层设计

`CAEProcessingFacade` 的职责是：

1. 初始化独立 OpenGL 计算上下文。
2. 加载数据集并建立内部缓存。
3. 按网格类型和方法分派梯度计算。
4. 组织多尺度数据优化流程。
5. 统一管理结果写回、汇总与导出。

该设计的好处是 GUI 与测试程序都只面向统一 API，而无需分别维护一套计算逻辑。

### 3.4 GUI 与测试程序设计

GUI 主窗口实现位于 [app/MainWindow.h](../app/MainWindow.h) 和 [app/MainWindow.cpp](../app/MainWindow.cpp)，主要负责：

1. 文件加载与导出。
2. 算法参数录入。
3. 调用门面层接口。
4. 基于 VTK 的结果显示。

测试程序则负责：

## 第四章 算法

### 4.1 规则网格有限差分算法

规则网格梯度算法流程如下：

1. 根据点场或单元场确定逻辑维度。
2. 在每个逻辑方向选择中心差分或单边差分模板。
3. 计算参数空间导数。
4. 构造局部 Jacobian。
5. 将参数导数映射为物理空间梯度。

该算法适用于 [Data/SampleStructGrid.vtk](../Data/SampleStructGrid.vtk) 等规则网格数据。

### 4.2 非结构化网格形函数导数算法

#### 4.2.1 点数据梯度

对点数据，系统按“样本点所在关联单元集合”进行局部重构。对每个关联单元：

1. 读取单元类型与节点坐标。
2. 在该单元的参考坐标系中获得节点形函数导数。
3. 构造 Jacobian 或度量张量。
4. 将参数导数映射到物理空间。
5. 对一个点所关联的多个单元结果进行汇聚。

当前点梯度主着色器为 [Shaders/ShapePointGradient.glsl](../Shaders/ShapePointGradient.glsl)。若该路径返回全零缓冲区，则代码中会回退到“先算单元梯度，再提升到点”的保底路径，相关逻辑在 [GLGradientEngine.cpp](../GLGradientEngine.cpp)。

#### 4.2.2 单元数据梯度

对单元数据，当前实现采用两步法：

1. 先将单元场通过 [Shaders/CellDataToPointLift.glsl](../Shaders/CellDataToPointLift.glsl) 提升为点场；
2. 再在单元中心位置通过 [Shaders/ShapeCellGradient.glsl](../Shaders/ShapeCellGradient.glsl) 计算单元梯度。

这一设计保证点数据和单元数据最终都回到统一的“形函数导数 + 局部映射”框架下。

#### 4.2.3 支持的单元类型

当前着色器中实现的单元包括：

- 线单元
- 三角形单元
- 像素/四边形单元
- 四面体单元
- 体素/六面体单元
- 棱柱单元

其中，本文正式实验口径建议聚焦：

- 一阶三角形
- 一阶四边形
- 一阶四面体
- 一阶六面体

### 4.3 自适应加权最小二乘算法

系统保留了 AWLS 实现，其流程包括：

1. 构造点邻域或单元邻域。
2. 根据局部几何特征估计维度标签与质量信息。
3. 组装最小二乘法方程组。
4. 在 GPU 上求解梯度。

由于本文主线转向形函数导数法，因此 AWLS 可在论文中作为扩展实现或对照算法简述，不作为主结论来源。

### 4.4 多尺度数据优化算法

多尺度优化流程如下：

1. 从点邻域或单元邻域构造图。
2. 估计平均邻距和字段标准差。
3. 按尺度递增进行图双边滤波。
4. 由相邻平滑层求细节层。
5. 按设定增益融合细节与基础层，得到最终输出。

中间结果命名规则在 [CAEProcessingFacade.cpp](../CAEProcessingFacade.cpp) 中定义，包括：

- `*_ms_s1_*`、`*_ms_s2_*`、`*_ms_s3_*`
- `*_ms_d0_*`、`*_ms_d1_*`、`*_ms_d2_*`
- `*_ms_base_*`
- `*_ms_fused_*`

#### 5.3.4 文献测试口径与本文实验组织的对应关系

结合第二章引用的文献，梯度计算方法的常见测试组织方式可概括为四类：一是制造解或解析解下的误差与阶次验证；二是不同单元类型、网格扰动、曲率和长宽比条件下的敏感性分析；三是真实工程字段与成熟软件结果的一致性对照；四是在明确计算条件后的性能比较。本文的实验设计与之对应如下：

1. 在规则网格 `SampleStructGrid` 以及非结构网格 `hexa`、`limb`、`ShipHull_0` 上注入 `benchmark_*` 场，与 `*_exact_grad` 比较，用于承担“算法正确性验证”的角色。
2. 通过体网格 `hexa`、复杂体网格 `limb` 和曲面网格 `ShipHull_0` 的分组测试，承担“单元类型与几何形态敏感性分析”的角色；其中 `ShipHull_0` 进一步使用 `benchmark_surface_*` 区分 ambient 指标与 intrinsic 指标，以避免把曲面法向分量误差直接混入主结论。
3. 通过真实字段与 `vtkGradientFilter` 的对照实验，承担“工程一致性验证”的角色；其结论只描述与 VTK 的接近程度，不替代真值验证。
4. 通过分别记录 VTK 顺序/并行时间、本系统墙钟时间和 GPU 纯计算时间，承担“性能比较”的角色；其中并行配置必须与后端和线程数一起报告。

需要指出的是，完整文献中常见的“多级网格加密后观察形式收敛阶”属于更标准但也更重的验证方式。本文当前主线已优先完成解析场误差验证、真实字段对照和时间性能评估；若后续时间允许，可在 `SampleStructGrid` 或 `hexa` 上补充三层以上网格加密实验，作为对文献标准的进一步靠拢。

### 5.4 评价指标

当前测试程序已经实现并输出多种指标，定义分别见：

- 梯度指标实现：[TestGradient.cpp](../TestGradient.cpp)
- 数据优化与粗糙度指标实现：[TestMultiScale.cpp](../TestMultiScale.cpp), [TestHarnessUtils.h](../TestHarnessUtils.h)

其中，本文实验分析主要使用以下指标：

1. `NRMSE`：归一化均方根误差，用于衡量与参考梯度或干净场的整体偏差。
2. `SoftRel Mean`：软相对误差均值，用于避免参考量接近零时普通相对误差失稳。
3. `RMSE 改进率`：定义为 `fused_rmse / input_rmse`，小于 `1` 表示优化后优于输入，大于 `1` 表示未获得改善。
4. `粗糙度比`：定义为 `fused_roughness / input_roughness`，越小表示局部高频起伏抑制越明显。
5. `gl_wall_avg_ms` 与 `gl_gpu_avg_ms`：分别表示完整流程墙钟时间与 GPU 纯计算时间。

### 5.7 本章讨论

根据上述表格，可以得到以下实验结论：

1. 梯度模块在真实 CAE 字段上与 VTK 的工程一致性较好，足以作为系统可用性的主要证据。
2. 当前时间对比表明系统在现有 VTK 顺序后端配置下具有明显速度优势，但尚不能据此宣称“全面优于 VTK 并行实现”。
3. 解析场实验显示，规则网格与非结构网格点数据路径整体较稳定，而单元数据路径对恢复策略更敏感。
4. 对曲面类数据，应优先使用曲面专用解析 benchmark，而不宜直接使用普通三维 ambient 基准。
5. 数据优化模块对高斯型和混合型局部高频扰动具有稳定抑制能力，对脉冲型异常扰动不敏感，这与模块的目标定位一致。

## 第六章 结论

### 6.1 已完成工作总结

根据当前工程实现，可以形成如下阶段性结论：

1. 已完成基于门面层的数据处理系统设计，实现了 GUI、测试程序和底层 GPU 计算的统一组织。
2. 已完成规则网格有限差分梯度计算与非结构网格形函数导数梯度计算的 GPU 实现，正式实验口径可覆盖一阶三角形、四边形、四面体和六面体等代表性单元。
3. 在真实字段对比中，系统与 VTK 的结果整体接近：规则网格、六面体点场与 `limb` 单元场的 `NRMSE` 达到 `1e-6` 量级，`ShipHull_0` 的真实字段对比也保持在 `0.0039` 到 `0.0215` 的范围内。
4. 在当前 VTK 顺序后端配置下，系统相对 VTK 的墙钟时间加速比约为 `4.81x` 到 `257.32x`，说明 GPU 主路径已具备明显的时间优势。
5. 已完成图双边滤波驱动的多尺度数据优化模块，并在 `ShipHull_0` 的合成扰动实验中观察到整体 `NRMSE` 从 `0.407763` 降至 `0.334165`，其中对高斯型局部随机高频扰动的抑制最为明显。

### 6.2 当前限制

当前限制主要包括：

1. 非结构网格单元数据路径当前采用 `cell -> point lift -> shape gradient` 两步恢复策略，因此解析场误差显著高于点数据路径，这也是 `hexa cell`、`limb cell` 指标偏大的主要原因。
2. 曲面型数据 `ShipHull_0` 在普通三维 ambient 解析 benchmark 下误差较大，因此论文中不宜直接用这一路径支撑“高精度”结论，而应改用曲面专用 benchmark 或真实字段对比。
3. 当前测试环境中的 VTK 参考后端为 `Sequential`，尚未真正启用多线程并行后端，因此时间对比结论应限定在“当前构建与配置下”的范围内。
4. 数据优化模块对高斯型和混合型局部高频扰动有效，但对脉冲型异常扰动抑制能力有限，说明其仍属于面向特定扰动类别的稳定化模块，而不是通用去噪器。
5. 系统的 I/O 与渲染仍依赖 VTK，尚未形成完全 VTK-free 的完整软件栈。

1. 学习 `vtkGradientFilter` 的退化保护机制，改进复杂曲面数据上的点梯度稳定性。
2. 设计更一致的 `cell-to-point` 重建方法，以提升单元数据解析场下的正确性。
3. 在启用 `vtkSMPTools` 真正并行后端后重新完成 VTK 并行时间对比实验。
4. 为曲面数据增加更明确的切向/法向分解分析，并补充更多 ParaView 可视化对照图。
5. 进一步扩展数据优化模块的扰动模型与评价方式，例如引入更贴近真实 CAE 数值扰动来源的实验设计。
