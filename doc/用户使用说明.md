# 用户使用说明

## 1. 适用范围

本文档面向当前工程 `OpenGLDP` 的日常使用，覆盖：

1. 编译环境准备
2. 工程构建
3. GUI 使用流程
4. 梯度测试程序使用
5. 多尺度测试程序使用
6. 结果查看与常见问题

工程构建入口见 [CMakeLists.txt](../CMakeLists.txt)。

## 2. 环境准备

### 2.1 基本依赖

当前工程依赖：

1. CMake 3.21+
2. Visual Studio 2019 / MSVC
3. Qt 5.15
4. VTK 9.x
5. OpenGL 4.3+ 显卡驱动

当前 `CMakeLists.txt` 中已写入本地默认路径：

- `VTK_DIR`
- `GLAD_ROOT`

如果本机路径不同，需要在配置时显式修改。

### 2.2 输入数据格式

当前 GUI 和测试程序都使用 `vtkDataSetReader` 读取数据，因此推荐输入格式为：

- legacy VTK `.vtk`

数据目录示例见：

- [Data](../Data)

## 3. 编译方法

### 3.1 使用 Qt Creator 构建

推荐流程：

1. 用 Qt Creator 打开 [CMakeLists.txt](../CMakeLists.txt)。
2. 选择带 MSVC2019 的 Qt Kit。
3. 确认 `VTK_DIR` 和 `GLAD_ROOT` 配置正确。
4. 配置并构建工程。

构建后会生成以下目标：

- `opengldp_gui`
- `opengldp_benchmark`
- `opengldp_multiscale_test`
- `opengldp_field_metrics`

### 3.2 使用命令行构建

示例命令：

```powershell
cmake -S . -B build -DVTK_DIR="C:\path\to\vtk\lib\cmake\vtk-9.x" -DGLAD_ROOT="C:\path\to\glad"
cmake --build build --config Release
```

如果直接复用工程中的默认路径，也可以省略相应选项。

### 3.3 运行目录说明

构建后，CMake 会自动把以下目录复制到可执行文件输出目录：

1. `Shaders/`
2. `Data/`

因此建议：

- 在可执行文件所在目录运行测试程序；
- 或确保当前工作目录下能找到 `Shaders` 和 `Data`。

## 4. GUI 使用流程

GUI 主程序入口见：

- [app/main.cpp](../app/main.cpp)
- [app/MainWindow.cpp](../app/MainWindow.cpp)

### 4.1 启动

运行：

```powershell
.\opengldp_gui.exe
```

若启动失败，优先检查：

1. `Shaders` 目录是否存在于程序运行目录。
2. 显卡驱动是否支持 OpenGL 4.3 Compute Shader。
3. Qt 与 VTK 运行库是否已正确部署。

### 4.2 加载数据

在 GUI 中点击 `Open VTK`，选择 `.vtk` 文件。当前推荐数据：

- [Data/SampleStructGrid.vtk](../Data/SampleStructGrid.vtk)
- [Data/hexa.vtk](../Data/hexa.vtk)
- [Data/ShipHull_0.vtk](../Data/ShipHull_0.vtk)
- [Data/limb.vtk](../Data/limb.vtk)

加载成功后：

1. 左侧列表会出现数据集名称。
2. 中间可切换 Point/Cell 字段。
3. 右侧可查看 VTK 渲染结果。

### 4.3 计算梯度

操作顺序：

1. 选择数据集。
2. 选择 `Association`：`Point` 或 `Cell`。
3. 选择输入字段。
4. 选择 `Method`：
   - `Auto`
   - `FiniteDifference`
   - `AdaptiveWLS`
   - `ShapeFunction`
5. 点击 `Compute Gradient`。

说明：

1. `Auto` 当前会把规则网格分派到 FD，把非结构网格分派到 Shape Function。
2. 计算结果会作为新字段写回当前数据集。
3. 日志窗口会显示结果字段名、墙钟时间和 GPU 时间。

### 4.4 数据优化

操作顺序：

1. 选择数据集和字段。
2. 设置 `MS Levels`、`MS Iter/Level`。
3. 设置 `Spatial Sigma Factor`、`Range Sigma Factor`、`Level Scale`、`Edge Sigma Factor`。
4. 设置 `Detail Gain L0/L1/L2`。
5. 点击 `Compute Multi-scale Optimization`。

输出结果通常包括：

- `*_ms_s1_*`
- `*_ms_s2_*`
- `*_ms_s3_*`
- `*_ms_d0_*`
- `*_ms_d1_*`
- `*_ms_d2_*`
- `*_ms_base_*`
- `*_ms_fused_*`

### 4.5 导出结果

点击 `Export Current Dataset` 后，可将当前数据集及新增字段导出为 legacy VTK 文件，便于在 ParaView 中查看。

## 5. 梯度测试程序使用

源码见 [TestGradient.cpp](../TestGradient.cpp)。

### 5.1 查看帮助

```powershell
.\opengldp_benchmark.exe --help
```

### 5.2 查看可用字段

```powershell
.\opengldp_benchmark.exe Data\ShipHull_0.vtk point --list-fields
```

### 5.3 规则网格测试示例

```powershell

```

### 5.4 非结构网格测试示例

```powershell
.\opengldp_benchmark.exe Data\hexa.vtk point --run=fields --reference=vtk --method=shape --csv=results\hexa_point.csv
```

### 5.5 解析基准场测试示例

```powershell
.\opengldp_benchmark.exe Data\hexa.vtk point --run=benchmarks --reference=analytic --method=shape --csv=results\hexa_bench.csv
```

### 5.6 输出解释

程序会输出：

1. 数据集信息
2. 几何摘要
3. 每个字段的运行时间
4. 与解析真值或 VTK 参考的误差指标
5. CSV 文件路径

## 8. 推荐使用流程

建议按以下顺序使用系统：

1. 先通过 GUI 确认数据能正常加载和显示。
2. 再通过 `opengldp_benchmark` 生成梯度定量结果。
3. 通过 `opengldp_multiscale_test` 生成数据优化结果和 ParaView 文件。
4. 必要时用 `opengldp_field_metrics` 对已生成字段做独立复核。
5. 最后在 ParaView 中查看导出的 `.vtk` 文件。

## 9. 常见问题

### 9.1 启动时提示 OpenGL 或着色器初始化失败

优先检查：

1. `Shaders` 目录是否存在。
2. 当前显卡驱动是否支持 OpenGL 4.3。
3. 是否从错误的工作目录启动程序。

### 9.2 GUI 能加载但测试程序找不到数据

原因通常是当前工作目录不在目标输出目录。解决方式：

1. 进入可执行文件目录再运行；
2. 或使用完整路径传入 `--dataset=`。

### 9.3 导出的 VTK 无法在 ParaView 打开

当前导出采用 legacy VTK。若遇到兼容性问题，优先确认：

1. 导出是否成功完成；
2. 是否使用了二进制导出；
3. 字段中是否存在异常值。

### 9.4 `notch_stress` 结果异常

当前 `notch_stress` 在点梯度路径上仍有已知稳定性问题，不建议作为主实验数据。可优先使用：

- `SampleStructGrid`
- `hexa`
- `ShipHull_0`
- `limb`
