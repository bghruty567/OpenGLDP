# OpenGLDP 系统使用说明

## 1. 当前版本的核心结论

当前版本的梯度模块已经定型为：

- 规则网格：有限差分 `FD`
- 非结构网格：统一自适应加权最小二乘 `AWLS`

另外需要特别注意两点：

- `results/` 目录里的旧结果不代表这次定稿结论。
- 解析场 benchmark 数组现在默认关闭，只在测试程序里通过开关显式启用，GUI 不使用它。

## 2. 当前支持的输入数据集

系统当前支持：

- `vtkImageData`
- `vtkRectilinearGrid`
- `vtkStructuredGrid`
- `vtkUnstructuredGrid`

内部自动归类为：

- 规则网格
- 非结构网格

## 3. 构建说明

项目使用：

- C++17
- CMake
- Qt5
- VTK 9.x
- OpenGL 4.3+

根目录的 `CMakeLists.txt` 已经把 `Shaders/` 和 `Data/` 复制到可执行程序目录。

## 4. GUI 怎么用

GUI 的主要流程如下：

1. 打开 `.vtk` 文件。
2. 选择 `Point` 或 `Cell`。
3. 选择数组。
4. 方法选择建议保持 `Auto`。
5. 点击梯度计算。
6. 在结果数组中查看 `*_grad_*_FD` 或 `*_grad_*_AWLS`。

当前 GUI 里的方法框实际上对应的是：

- `Auto`
- `FiniteDifference`
- `AdaptiveWLS`

其中：

- 对规则网格，`Auto` 会自动转成 `FD`
- 对非结构网格，`Auto` 会自动转成 `AWLS`

## 5. 测试程序怎么用

梯度验证程序是 `opengldp_benchmark`。

### 5.1 默认行为

默认情况下，它只对数据文件里原有的真实数组做计算和对比。

如果存在 `xxx_exact_grad`，测试程序会优先拿它作为参考真值；否则才会退回到 `vtkGradientFilter`。

### 5.2 解析场测试开关

只有显式传入下面的参数，程序才会把解析 benchmark 数组注入到数据集中：

```text
--analytic-bench
--analytic-benchmark
--analytic-bench=on
```

显式关闭可以用：

```text
--no-analytic-bench
--analytic-bench=off
```

### 5.3 建议的测试顺序

建议按下面顺序验证：

1. 先做解析场测试。
2. 再做真实数据与 `vtkGradientFilter` 的对照。
3. 最后做性能统计。

## 6. 指标怎么理解

测试程序当前输出的核心指标包括：

| 指标 | 含义 |
| --- | --- |
| `NMAE_vec` | 向量绝对误差的归一化平均值 |
| `NRMSE_vec` | 向量均方根误差的归一化值 |
| `Angle_mean_deg` / `Angle_P90_deg` | 方向误差 |
| `ScaleBias` | 整体尺度偏差 |
| `SoftRel_mean` / `SoftRel_P90` | 软相对误差，减轻低参考值区域放大效应 |

建议解释方式：

- 解析场测试里，`NMAE_vec < 1` 且 `NRMSE_vec < 1`，说明当前方案已经具备可用的主正确性证据。
- 真实数据对比里，如果某些复杂样例相对 `vtkGradientFilter` 仍然大于 1，只能说明还没有完全逼近 VTK，不等于算法失效。

## 7. 你当前这版结果该怎么写进论文

按照你现在的描述：

- 解析场测试在当前所有测试数据上 `NMAE_vec < 1`、`NRMSE_vec < 1`
- 真实数据里 `ShipHull_0` 和 `notch_stress` 相对 VTK 的这两个指标在 `1` 到 `2` 之间

我建议论文这样表述：

1. 当前方案可以作为本科毕设的最终工程方案。
2. 主要正确性依据应放在解析场 exact gradient 验证上。
3. 真实数据与 VTK 的差异应当写成“困难样例下仍存在偏差”。
4. 不要写成“当前方案已经在所有非结构网格上与 VTK 等价”。

## 8. 为什么 GUI 不使用解析 benchmark

这样做是为了把“实验验证功能”和“实际系统使用”分开：

- GUI 面向真实数据处理流程
- benchmark 注入只服务于测试和论文验证

这也是当前代码已经落实的设计。

## 9. 建议保留的论文实验

建议至少保留四类实验：

1. 解析场 exact gradient 验证
2. 真实数据对 `vtkGradientFilter` 的工程对照
3. 参数敏感性实验
4. CPU/GPU 时间统计
