# OpenGLDP 系统使用说明

## 1. 文档目的

本文档用于说明当前 `OpenGLDP` 毕设项目的运行环境、编译方法、GUI 使用流程、测试程序使用方式以及常见问题排查方法。文档内容以当前代码仓库中的实际实现为准。

---

## 2. 系统概述

`OpenGLDP` 是一个面向 CAE 数据预处理与可视化的桌面系统，当前主要支持以下功能：

- 读取 VTK legacy 数据文件（`.vtk`）。
- 将 VTK 数据转换为内部统一数据结构 `DataObject`。
- 对规则网格和非结构网格执行梯度计算。
- 对标量/向量场执行基于双边滤波的多尺度分解。
- 对多尺度细节层执行加权融合，生成优化后的结果场。
- 在 Qt + VTK 界面中查看当前数据集和数组。
- 将包含计算结果的新数据集重新导出为 VTK 文件。

当前项目中包含 3 个可执行目标：

- `opengldp_gui`：主界面程序。
- `opengldp_benchmark`：梯度模块测试程序。
- `opengldp_multiscale_test`：数据优化模块测试程序。

---

## 3. 运行环境与依赖

### 3.1 基本环境

- 操作系统：Windows
- 编译器：支持 C++17 的 MSVC
- 构建系统：CMake 3.21 及以上
- Qt：Qt5，至少包含 `Widgets` 和 `OpenGL`
- VTK：VTK 9.x，当前 `CMakeLists.txt` 默认使用 `VTK 9.5`
- OpenGL：显卡与驱动需支持 `OpenGL 4.3` 及以上

### 3.2 项目中的关键依赖

- `VTK`：负责数据读取、数据导出和界面渲染。
- `Qt5`：负责 GUI。
- `OpenGL + GLAD`：负责计算着色器执行。

### 3.3 与当前 `CMakeLists.txt` 相关的路径配置

当前工程在 `CMakeLists.txt` 中写了默认路径：

- `VTK_DIR` 默认尝试为  
  `C:/Users/lenovo/Desktop/bishe/vtk/vtk-install/lib/cmake/vtk-9.5`
- `GLAD_ROOT` 默认尝试为  
  `C:/Users/lenovo/Desktop/bishe/opengl/glad4.6`

如果你的本机路径不同，需要在配置 CMake 时手动改成你自己的路径。

## 5. 编译与构建

### 5.1 推荐构建方式

在项目根目录执行：

```powershell
cmake -S . -B build `
  -DVTK_DIR="C:\你的VTK路径\lib\cmake\vtk-9.5" `
  -DGLAD_ROOT="C:\你的glad路径"
```

然后编译：

```powershell
cmake --build build --config Release
```

### 5.2 编译后目标

编译成功后，通常会得到以下程序：

- `build\...\opengldp_gui.exe`
- `build\...\opengldp_benchmark.exe`
- `build\...\opengldp_multiscale_test.exe`

### 5.3 关于 `Shaders` 和 `Data`

`CMakeLists.txt` 中已经写了 `POST_BUILD` 复制逻辑，会把：

- `Shaders/`
- `Data/`

复制到各个目标程序所在目录。

因此最稳妥的运行方式是：

- 在生成目录中直接运行可执行文件；
- 或者把当前工作目录设置在项目根目录，使相对路径 `Shaders`、`Data` 能被找到。

## 6. GUI 使用说明

### 6.1 启动程序

运行 `opengldp_gui.exe` 后，系统会先初始化：

- OpenGL 上下文
- 计算着色器
- Qt 主界面
- VTK 渲染窗口

如果初始化失败，通常会弹出提示框，日志区也会显示失败信息。

### 6.2 界面区域说明

主窗口大致分为三部分：

1. 左侧面板  
   负责打开文件、导出文件、查看数据集列表和摘要信息。

2. 中间参数面板  
   负责梯度计算参数和数据优化参数设置。

3. 右侧显示区域  
   上半部分是 VTK 渲染窗口，下半部分是日志输出区域。

### 6.3 数据加载

点击 `Open VTK` 按钮后：

1. 选择一个 `.vtk` 文件。
2. 程序调用 `CAEProcessingFacade::loadDatasetFromVTKFile(...)`。
3. 数据会被转换为内部 `DataObject`。
4. 左侧数据集列表会新增一条记录。
5. 当前数据集对应的字段列表会自动刷新。

推荐先使用 `Data/` 目录中的样例：

- `Data\hexa.vtk`
- `Data\SampleStructGrid.vtk`
- `Data\blow.vtk`
- `Data\ShipHull_0.vtk`

### 6.4 梯度计算模块使用方法

中间区域上半部分是梯度计算参数。

#### 参数说明

| 控件 | 作用 |
| --- | --- |
| `Association` | 选择当前操作的是点数据还是单元数据 |
| `Array` | 选择要计算梯度的输入数组 |
| `Method` | 选择 `Auto / FiniteDifference / WeightedLeastSquares` |
| `WLS Exponent` | 非结构网格 WLS 距离权重指数 |
| `WLS Lambda` | 非结构网格 WLS 正则参数 |
| `Visible Component` | 控制当前渲染的数组分量 |

#### 使用步骤

1. 加载数据文件。
2. 在 `Association` 中选择点数据或单元数据。
3. 在 `Array` 中选择一个要处理的数组。
4. 在 `Method` 中选择方法：
   - `Auto`：规则网格自动走 FD，非结构网格自动走 WLS。
   - `FiniteDifference`：规则网格梯度。
   - `WeightedLeastSquares`：非结构网格梯度。
5. 点击 `Compute Gradient`。
6. 日志区会输出结果数组名、CPU 耗时、GPU 耗时。
7. 新生成的梯度数组会自动加入当前数据集。

#### 结果命名

梯度结果命名格式为：

```text
<源数组名>_grad_<P或C>_<FD或WLS>
```

例如：

- `pressure_grad_P_FD`
- `temperature_grad_C_WLS`

### 6.5 数据优化模块使用方法

中间区域下半部分是数据优化模块，也就是你现在实现的“基于双边滤波的多尺度分解 + 基于加权的多尺度融合”。

#### 参数说明

| 控件 | 作用 |
| --- | --- |
| `MS Levels` | 分解层数，当前界面限制为 1~3 层 |
| `MS Iter/Level` | 每层双边滤波迭代次数 |
| `Spatial Sigma Factor` | 空间尺度系数，乘以平均邻接距离得到 `spatialSigma` |
| `Range Sigma Factor` | 值域尺度系数，乘以字段标准差得到 `rangeSigma` |
| `Level Scale` | 层间尺度放大倍数 |
| `Edge Sigma Factor` | 融合阶段边缘抑制系数 |
| `Detail Gain L0` | 第 0 层细节增益 |
| `Detail Gain L1` | 第 1 层细节增益 |
| `Detail Gain L2` | 第 2 层细节增益 |
| `Store intermediate smooth/detail arrays` | 是否把各层平滑结果与细节结果回写到数据集中 |

#### 使用步骤

1. 先加载一个数据集。
2. 在 `Association` 和 `Array` 中选定输入数组。
3. 设置多尺度参数。
4. 点击 `Run MultiScale Optimization`。
5. 程序会执行：
   - 邻接图构建；
   - 多层图双边滤波；
   - 细节层计算；
   - 多尺度加权融合；
   - 结果数组回写。
6. 日志区会显示源数组、融合结果数组、层数、CPU/GPU 时间，以及每层平滑/细节数组名。

#### 结果命名

平滑层：

```text
<源数组名>_ms_s<层号>_<P或C>
```

细节层：

```text
<源数组名>_ms_d<层号>_<P或C>
```

融合基底：

```text
<源数组名>_ms_base_<P或C>
```

融合结果：

```text
<源数组名>_ms_fused_<P或C>
```

例如：

- `pressure_ms_s1_P`
- `pressure_ms_d0_P`
- `pressure_ms_base_P`
- `pressure_ms_fused_P`

### 6.6 导出结果

点击 `Export Current Dataset` 后：

1. 选择导出路径；
2. 系统会把当前 `DataObject` 重新转换成 `vtkDataSet`；
3. 原始数组与新增数组会一起写回 `.vtk` 文件。

适用于：

- 保存梯度结果；
- 保存多尺度分解结果；
- 保存融合结果；
- 后续在 ParaView 或其他软件中继续分析。

## 7. 测试程序使用说明

### 7.1 梯度模块测试程序 `opengldp_benchmark`

该程序对应源码 `TestGradient.cpp`，用于：

- 测试门面接口是否正常；
- 对比 OpenGL 结果与 `vtkGradientFilter` 结果；
- 输出 CPU/GPU 时间；
- 输出误差指标，如 MAE、RMSE、MAX。

#### 命令格式

```powershell
opengldp_benchmark [vtk文件路径] [point|cell] [数组名] [重复次数]
```

#### 参数说明

| 参数 | 含义 | 默认值 |
| --- | --- | --- |
| `vtk文件路径` | 输入数据文件 | `Data\hexa.vtk` |
| `point|cell` | 选择点数据或单元数据 | `point` |
| `数组名` | 输入数组名，留空则默认取第一个数组 | 第一个可用数组 |
| `重复次数` | 性能测试重复次数 | `5` |

#### 示例

```powershell
opengldp_benchmark Data\hexa.vtk point Pressure 5
```

#### 输出内容

程序典型输出包括：

- 数据集摘要
- 可用数组列表
- 实际采用的数组名
- `VTK_time_ms_avg`
- `GL_wall_ms_avg`
- `GL_gpu_ms_avg`
- `MAE_abs`
- `RMSE`
- `MAX_abs`

### 7.2 数据优化模块测试程序 `opengldp_multiscale_test`

该程序对应源码 `TestMultiScale.cpp`，用于：

- 测试多尺度分解与融合是否执行成功；
- 统计优化前后的标准差变化；
- 统计图粗糙度变化；
- 导出带有中间结果和融合结果的 `.vtk` 文件。

#### 命令格式

```powershell
opengldp_multiscale_test [vtk文件路径] [point|cell] [数组名] [重复次数] [层数] [每层迭代次数] [导出路径]
```

#### 参数说明

| 参数 | 含义 | 默认值 |
| --- | --- | --- |
| `vtk文件路径` | 输入数据文件 | `Data\SampleStructGrid.vtk` |
| `point|cell` | 点数据或单元数据 | `point` |
| `数组名` | 输入数组名，留空默认取第一个 | 第一个可用数组 |
| `重复次数` | 性能重复次数 | `3` |
| `层数` | 多尺度层数，程序内部会限制到 1~3 | `3` |
| `每层迭代次数` | 每层双边滤波迭代次数 | `1` |
| `导出路径` | 导出带结果的 VTK 文件路径 | `results\multiscale_out.vtk` |

#### 示例

```powershell
opengldp_multiscale_test Data\SampleStructGrid.vtk point pressure 3 3 1 results\multiscale_out.vtk
```

#### 输出内容

程序典型输出包括：

- `MS_wall_ms_avg`
- `MS_gpu_ms_avg`
- `InputStd / FusedStd`
- `MeanAbsDelta`
- `InputRoughness / FusedRoughness`
- `BaseArray`
- `FusedArray`
- `Smooth[i]`
- `Detail[i]`
- `Exported=...`

#### 结果解释建议

- `FusedStd < InputStd`：通常说明噪声或局部起伏有所减弱。
- `FusedRoughness < InputRoughness`：通常说明邻域间变化更平滑。
- `MeanAbsDelta`：表示优化前后整体改变量大小。

## 8. 建议的使用流程

如果你是写毕设、截图和做实验，建议按照下面顺序使用：

1. 先用 `opengldp_gui` 观察数据和结果是否符合预期。
2. 用 `opengldp_benchmark` 做梯度模块定量测试。
3. 用 `opengldp_multiscale_test` 做数据优化模块定量测试。
4. 用 `Export Current Dataset` 或测试程序导出 `.vtk`。
5. 在 ParaView 或系统 GUI 中再次查看结果场。

## 9. 常见问题与排查

### 9.1 程序启动即提示初始化失败

优先检查：

- 显卡驱动是否支持 `OpenGL 4.3+`
- 当前目录下是否存在 `Shaders`
- `FD.glsl`、`WLS.glsl`、`Bilateral.glsl`、`MultiScaleFuse.glsl` 是否完整

### 9.2 打开文件后看不到数组

可能原因：

- 当前 `Association` 选成了 `Cell`，但数据主要是点数据；
- 或者反过来，当前选成了 `Point`，但数组在单元上。

### 9.3 点击计算按钮没有结果

检查以下项目：

- 是否已加载数据集；
- 是否已经选中输入数组；
- 输入数组是否为空；
- 对规则网格是否错误强制使用了不适合的方法；
- 对非结构网格是否邻接构建失败。

### 9.4 多尺度结果没有中间数组

这通常是因为没有勾选 `Store intermediate smooth/detail arrays`。

### 9.5 程序运行时找不到 `Shaders` 或 `Data`

说明当前工作目录不对。解决方式：

- 在生成目录中运行程序；
- 或者把程序工作目录改成项目根目录；
- 或者确认 `POST_BUILD` 复制是否成功。
