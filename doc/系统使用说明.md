# OpenGLDP 系统使用说明

## 1. 适用范围

当前项目主要提供三类可执行目标：

- `opengldp_gui`：图形界面程序；
- `opengldp_benchmark`：梯度测试与对照程序；
- `opengldp_multiscale_test`：多尺度模块测试程序。

其中，本次更新重点涉及 `opengldp_benchmark`。

## 2. 构建说明

项目当前使用：

- C++17
- CMake
- Qt5
- VTK 9.x
- OpenGL 4.3+

构建后，`Shaders/` 和 `Data/` 会被复制到对应可执行程序目录。

## 3. GUI 基本使用

GUI 的典型流程如下：

1. 打开 `.vtk` 数据集；
2. 选择 `Point` 或 `Cell`；
3. 选择输入数组；
4. 方法建议保持 `Auto`；
5. 执行梯度计算；
6. 在结果数组中查看生成的梯度字段。

当前 `Auto` 的实际含义是：

- 规则网格自动走 `FD`；
- 非结构网格自动走 `AdaptiveWeightedLeastSquares`。

## 4. `TestGradient` 的定位

新的 `TestGradient` 不再只是一个单一案例脚本，而是一个统一实验入口，用于：

- 跑单个字段；
- 一次性跑全部 benchmark；
- 一次性跑全部字段；
- 输出 CSV 汇总；
- 在解析梯度与 VTK 参考之间切换。

## 5. `TestGradient` 常用参数

| 参数 | 含义 |
| --- | --- |
| `--dataset=<path>` | 数据集路径 |
| `--assoc=point|cell` | 选择点数据或单元数据 |
| `--array=<name>` | 单案例模式下指定输入数组 |
| `--reps=<n>` | 重复次数，用于计时统计 |
| `--reference=auto|analytic|vtk|none` | 参考来源 |
| `--analytic-bench=on|off` | 是否注入解析 benchmark |
| `--dump=<n>` | 单案例模式下打印前 n 个结果 |
| `--run=single|benchmarks|fields` | 运行模式 |
| `--filter=<text>` | 仅保留名字匹配的字段 |
| `--list-fields` | 列出当前关联类型下的全部数组 |
| `--list-benchmarks` | 列出当前可用 benchmark |
| `--csv=<path>` | 导出 CSV 汇总 |
| `--show-config` | 打印解析后的配置 |

保留的旧式位置参数仍可使用：

```text
opengldp_benchmark [dataset] [point|cell] [array] [reps]
```

## 6. 新版 benchmark 能力

当前测试程序在开启解析 benchmark 时，会自动注入以下字段到点数据和单元数据两侧：

### 6.1 标量 benchmark

- `benchmark_linear`
- `benchmark_quadratic`
- `benchmark_trig`
- `benchmark_cubic`
- `benchmark_gaussian`

对应精确梯度分别为：

- `benchmark_linear_exact_grad`
- `benchmark_quadratic_exact_grad`
- `benchmark_trig_exact_grad`
- `benchmark_cubic_exact_grad`
- `benchmark_gaussian_exact_grad`

### 6.2 向量 benchmark

- `benchmark_vec_linear`
- `benchmark_vec_poly`
- `benchmark_vec_trig`

对应精确梯度分别为：

- `benchmark_vec_linear_exact_grad`
- `benchmark_vec_poly_exact_grad`
- `benchmark_vec_trig_exact_grad`

## 7. 推荐使用方式

### 7.1 解析 benchmark 套件

用于验证算法主路径是否正确：

```text
opengldp_benchmark --dataset=Data\AngularSector.vtk --assoc=cell --run=benchmarks --reference=analytic --csv=results\angular_cell_bench.csv
```

### 7.2 真实数据与 VTK 对照

用于做工程基线比较：

```text
opengldp_benchmark --dataset=Data\ShipHull_0.vtk --assoc=point --run=fields --reference=vtk --analytic-bench=off --csv=results\ship_point_vtk.csv
```

### 7.3 单案例调试

用于快速排查某个字段：

```text
opengldp_benchmark Data\AngularSector.vtk cell benchmark_quadratic 5 --reference=analytic --dump=20
```

### 7.4 先列字段再跑

```text
opengldp_benchmark --dataset=Data\AngularSector.vtk --assoc=cell --list-fields
opengldp_benchmark --dataset=Data\AngularSector.vtk --assoc=cell --list-benchmarks
```

## 8. 输出结果如何理解

程序会输出：

- 数据集信息；
- 选中的字段列表；
- OpenGL 侧 CPU/GPU 时间；
- 参考来源；
- 误差指标；
- 可选的样本级输出；
- 可选的 CSV 汇总。

需要特别注意：

- `reference=analytic` 时，参考是真正的解析梯度；
- `reference=vtk` 时，参考是 `vtkGradientFilter`；
- `reference=auto` 时，优先用解析梯度，找不到再退回到 VTK；
- `reference=none` 时，只做计算与计时，不做对照。

## 9. 推荐的测试顺序

建议固定按下面顺序做实验：

1. `--run=benchmarks --reference=analytic`  
   先证明算法本身正确。
2. `--run=fields --reference=vtk`  
   再看真实数据对 VTK 的工程表现。
3. 必要时配合 `--filter=`、`--array=`、`--dump=` 做局部调试。
