# 文献支撑

## 1. 本文档的定位

这份文档只为当前最终方案服务，不再为旧的 patch recovery、局部稀疏梯度算子或基于 `vtkCell::Derivatives(...)` 的默认路线背书。

当前最终方案是：

- 规则网格：`FD`
- 非结构网格：统一 `AWLS`

另外需要明确：

- `results/` 目录里的旧结果不作为本轮结论依据。
- 当前结论以你这轮手工记录的测试结果为准。

## 2. 这版方案和文献到底是什么关系

最准确的说法不是“完全照搬某一篇论文”，而是：

> 当前方法以非结构网格加权最小二乘梯度重构文献为理论基础，在此基础上结合本系统的数据结构、GPU 执行方式和稳定性需求，做了统一化、工程化实现。

也就是说：

- 理论核心来自文献中的 WLS/LSQ 梯度重构；
- 自适应扩邻域、局部维数识别、质量驱动正则增强，属于工程增强项；
- 因而它是“按参考文献路线实现并工程化改造”的方案，而不是逐句复刻某一篇论文的代码。

## 3. 关键文献 1：Wang et al. (2019)

文献信息：

- Nianhua Wang, Ming Li, Rong Ma, et al.
- *Accuracy analysis of gradient reconstruction on isotropic unstructured meshes and its effects on inviscid flow simulation*
- *Advances in Aerodynamics*, 2019
- 链接：[Springer PDF](https://link.springer.com/content/pdf/10.1186/s42774-019-0020-9.pdf)

### 3.1 这篇文献支持了什么

它直接支持“在非结构网格上采用 LSQ/WLS 梯度重构是合理的”。

### 3.2 可以直接引用的页码与句子

下面这些位置建议你在论文里直接引用或转述：

1. PDF 第 1 页摘要部分，含有短语 `"at least first-order"`。
   - 含义：LSQ 方法在任意非结构网格上至少是一阶。

2. PDF 第 2 页前言部分，说明一阶准确梯度对于二阶离散是必要条件。
   - 含义：梯度重构质量会直接影响后续数值结果。

3. PDF 第 17 页数值实验部分，作者把计算梯度与 exact gradient 做 `L1` 和 `L2` 范数误差比较。
   - 含义：解析场 exact gradient 验证是标准、扎实的验证方法。

4. PDF 第 19 页，文中出现短语 `"lowest level of error"`，用于描述 extended WLSQ 在其测试中的误差表现。
   - 含义：扩大支撑、改善 stencil 选择，本身就有文献基础。

### 3.3 当前代码使用了文献中的哪个公式

当前方案与这篇文献最相关的是其关于 LSQ 重构的推导：

- 式 `(17)`：局部线性重构关系
- 式 `(18)`：最小二乘目标函数
- 式 `(19)`：矩阵形式的 LSQ 解
- 式 `(20)`：在各向同性非结构网格上的一阶结论

当前代码没有逐字照搬论文符号，但本质上做的是同类事情：

1. 在局部邻域上建立线性梯度模型；
2. 在局部邻域上最小化残差；
3. 输出每个样本点或单元中心的梯度。

### 3.4 它对应到当前代码哪里

对应关系如下：

| 文献思想 | 当前代码 |
| --- | --- |
| 局部 stencil 上的 LSQ 梯度重构 | `GLGradientEngine::computeUnstructuredAdaptiveWLS(...)` |
| 扩展支撑、改善 stencil | `ensureAdaptiveSupport(...)` 中的扩二环与 KNN 补邻域 |
| 梯度精度需要用 exact gradient 验证 | `TestGradient.cpp` 中优先读取 `*_exact_grad` |

## 4. 关键文献 2：Mavriplis & Thomas (2003)

文献信息：

- Dimitri J. Mavriplis, James L. Thomas
- *Revisiting the Least-squares Procedure for Gradient Reconstruction on Unstructured Meshes*
- NASA/CR-2003-212683, 2003
- 链接：[NASA PDF](https://ntrs.nasa.gov/api/citations/20040070704/downloads/20040070704.pdf)

### 4.1 这篇文献支持了什么

这篇文献一方面支持“加权最小二乘本身是标准路线”，另一方面也提醒你“它在困难网格上可能退化，所以不能神化”。

### 4.2 可以直接引用的页码与句子

建议重点用下面几处：

1. PDF 第 5 页，含有短语 `"represent a linear function exactly"`。
   - 含义：LSQ 梯度重构至少应满足线性精确性。

2. PDF 第 5 页，含有短语 `"much more important"`，上下文强调权重选择的重要性。
   - 含义：权重和 stencil 质量对结果很敏感。

3. PDF 第 13 页，含有短语 `"accuracy similar"`。
   - 含义：在合适处理后，困难情形下也可以恢复到可接受精度。

4. PDF 第 17 页，含有短语 `"poor accuracy"`。
   - 含义：在某些高拉伸、困难网格情形下，LSQ 路线确实可能表现不佳。

### 4.3 当前代码使用了文献中的哪个公式

当前实现与其附录中关于加权最小二乘的推导最相关：

- 式 `(29)` 到 `(35)`：带权 LSQ 的正规方程推导

这部分和当前实现的关系是：

- 当前 `WLS.glsl` / `AdaptiveWLS.glsl` 都属于这类带权局部最小二乘求解；
- 代码里的 `wlsExponent` 对应距离权重的幂指数思想；
- 代码里的 `wlsLambda` 与 `lambdaAmplify` 对应工程上的正则稳定化处理。

### 4.4 它对应到当前代码哪里

| 文献思想 | 当前代码 |
| --- | --- |
| 带权局部最小二乘 | `WLS.glsl` / `AdaptiveWLS.glsl` |
| 权重选择会显著影响结果 | `CAEGradientRequest::wlsExponent` |
| 困难网格上精度可能退化 | `ShipHull_0`、`notch_stress` 这类样例的局限解释 |

## 5. 当前方法到底用了哪一种方法

如果你要在论文里只写一句话，建议写：

> 本文在非结构网格上采用基于局部支撑的自适应加权最小二乘梯度重构方法。

更具体一点可以写成：

1. 样本位置取网格点或单元中心；
2. 以点邻接或单元邻接作为基础 stencil；
3. 通过扩二环和 KNN 补齐支撑；
4. 通过局部协方差谱分析得到 `frame / dimTag / quality`；
5. 在 GPU 上执行带权最小二乘求解；
6. 对低质量邻域使用更强正则。

## 6. 为什么解析场结果比真实数据对 VTK 更重要

从论文验证角度，解析场 exact gradient 更接近“真值验证”。

这也是为什么 Wang 2019 会在数值实验里直接把重构梯度和 exact gradient 做误差比较。

所以你现在的结论应当这样排序：

1. 第一证据：解析场 exact gradient 验证。
2. 第二证据：真实数据相对 `vtkGradientFilter` 的工程对照。

## 7. 为什么 `ShipHull_0` 和 `notch_stress` 仍可能大于 1

这不自动说明当前方案不能定稿，原因有三点：

1. `vtkGradientFilter` 不是解析真值，只是参考实现。
2. 这两类样例通常几何更复杂、网格质量更差、局部拉伸更明显。
3. 文献本身就指出，LSQ 类方法在困难网格上会出现精度退化。

因此更合理的论文写法是：

- 这两组数据说明当前统一 AWLS 在复杂真实样例上仍有偏差；
- 但这不否定其作为本科毕设最终工程方案的合理性；
- 前提是你把主要正确性证据放在解析场 exact gradient 上。

## 8. 能不能把这版作为最终方案

结合你目前给出的测试现象：

- 解析场测试：当前所有测试数据 `NMAE_vec < 1`、`NRMSE_vec < 1`
- 真实数据测试：`ShipHull_0`、`notch_stress` 相对 VTK 的两项指标大于 1 但小于 2

我的判断是：

> 可以把这版作为本科毕设的最终工程方案。

但必须带着三个限定：

1. 不能宣称“已经在所有非结构网格上完全等价于 VTK”。
2. 需要明确说明解析场验证是主证据。
3. 需要把 `ShipHull_0`、`notch_stress` 写成困难样例和后续优化方向。

## 9. 论文中建议直接采用的表述

建议你在论文里使用下面这类表述：

> 本文最终采用规则网格有限差分与非结构网格自适应加权最小二乘相结合的梯度计算方案。非结构网格部分基于局部支撑上的加权最小二乘梯度重构思想实现，并结合局部支撑扩展、局部维数识别和质量驱动正则增强，以适应复杂非结构网格上的工程计算需求。解析场 exact gradient 验证表明该方案在当前测试数据上具备稳定的误差表现；对复杂真实样例，与 VTK 参考结果相比仍存在一定偏差，这部分作为本文方案的局限与后续工作保留。

## 10. 后续实验方案

建议把后续实验明确拆成四组。

### 10.1 实验 A：解析场正确性验证

目的：

- 证明算法本身是否正确

做法：

1. 对点数据和单元数据分别注入 `benchmark_linear`
2. 注入 `benchmark_quadratic`
3. 注入 `benchmark_trig`
4. 使用 `*_exact_grad` 直接计算 `NMAE_vec`、`NRMSE_vec`、角度误差

建议结论：

- 这是主正确性实验

### 10.2 实验 B：真实数据工程对照

目的：

- 评估和 `vtkGradientFilter` 的差异

做法：

1. 选择 `ShipHull_0`
2. 选择 `notch_stress`
3. 选择其余典型数据
4. 统计 `NMAE_vec`、`NRMSE_vec`、`Angle_P90_deg`、`ScaleBias`

建议结论：

- 这是工程对照实验，不是真值实验

### 10.3 实验 C：参数消融

目的：

- 看当前工程增强项到底有没有价值

建议消融项：

- 关闭 `useAdaptiveNeighborhood`
- 关闭 `useAdaptiveDimension`
- 关闭 `useAdaptiveRegularization`
- 改变 `wlsExponent`
- 改变 `wlsLambda`

### 10.4 实验 D：性能统计

目的：

- 证明 GPU 路线在工程上值得采用

建议统计：

- `VTK_time_ms_avg`
- `GL_wall_ms_avg`
- `GL_gpu_ms_avg`
