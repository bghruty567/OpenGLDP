# 梯度模块时间实验数据说明

## 1. 使用口径

这组数据只用于梯度模块的 `VTK` 时间对比实验，不用于替代真实数据精度对比实验。

为了保证控制变量成立，时间实验只保留两组网格族：

1. `规则网格族`
2. `代表性非结构化网格族`

其中“代表性非结构化网格族”统一固定为 `HEXAHEDRON` 六面体非结构化网格，不再同时混入 `TETRA`、`TRIANGLE` 或 `QUAD`。这样可以避免把“单元类型差异”误写成“规模差异”。

## 3. 数据设计

所有数据都满足以下控制条件：

- 几何域统一为 `[0,1]^3`
- 网格尺寸只在 `x/y/z` 三个方向同步放大
- 输入字段统一为 `scalars = 1.5x - 2.0y + 0.75z + 0.1`
- 同时写入 `POINT_DATA` 和 `CELL_DATA`
- 文件格式统一为 VTK legacy ASCII，便于现有程序直接读取

## 4. 推荐实验写法

### 4.1 实验目的

该实验用于比较不同实现路径在相同网格族、相同输入场和相同硬件环境下的处理时间，不承担算法正确性证明任务。

### 4.2 自变量

- 网格族：规则网格 / 非结构化六面体网格
- 规模：小 / 中 / 大

### 4.3 控制变量

- 几何域
- 输入字段
- 关联方式
- 重复次数
- 硬件环境
- 编译配置
- `VTK` backend 与线程设置

### 4.4 因变量

- `vtk_single_avg_ms`
- `vtk_parallel_avg_ms`
- `vtk_parallel_threads`
- `gl_wall_avg_ms`
- `gl_gpu_avg_ms`

## 5. 推荐命令

建议统一采用 `Release` 版本测试，并固定 `POINT + scalars` 作为主时间实验口径。

规则网格示例：

```text
opengldp_benchmark --dataset=Data\\timing\\timing_struct_48x48x48.vtk --assoc=point --array=scalars --run=single --reference=vtk --analytic-bench=off --reps=5
```

非结构化六面体示例：

```text
opengldp_benchmark --dataset=Data\\timing\\timing_uhex_48x48x48.vtk --assoc=point --array=scalars --run=single --reference=vtk --analytic-bench=off --reps=5
```

如果后续需要补充 `CELL` 时间实验，可仅把 `--assoc=point` 改为 `--assoc=cell`，其余条件保持不变。
