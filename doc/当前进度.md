# 基于 OpenGL 的 CAE 数据预处理系统设计方案与当前进展

## 1 课题背景与目标概述

本课题面向工程仿真后处理阶段，对高效率、高精度数据准备的需求，针对 CAE 软件输出的结构仿真与多物理场数据（网格模型、应力场、温度场等）在以下方面的特点：

- 数据规模大、维度高；
- 网格类型复杂（规则网格 + 各类非结构化网格）；
- 物理场分布不均匀，噪声较多，对渲染非常敏感。

课题核心思路：

- 构建 **面向 OpenGL 渲染管线的预处理框架**；
- 将 **梯度场计算、数据降噪、多尺度融合、特征保持平滑** 等前处理阶段算法迁移到 **GPU/渲染管线前端**；
- 通过 **VBO/SSBO + 计算着色器** 的方式实现并行加速；
- 提供统一的 **数据接口和 API 层**，为颜色映射、等值线/面、粒子渲染等可视化模块提供基础。

### 1.1 技术目标

- 面向 OpenGL 渲染管线，设计一套 CAE 数据预处理体系；
- 支持大规模网格、应力场、温度场等结构仿真数据的高效清洗、重构和优化；
- 基于有限差分（FD）、加权最小二乘（WLS）实现 **梯度场快速求解**；
- 结合数据降噪、尺度压缩、多尺度融合、特征保持平滑，提升数据连续性与渲染适配性。

### 1.2 功能目标

- 实现以下核心预处理功能：
  - 梯度场生成（规则/非结构化网格）；
  - 数据降噪（如双边滤波）；
  - 多尺度数据构建与融合；
  - 特征保持平滑；
  - 异常值检测与抑制（如孤立异常、极端值）。
- 支持物理场结果映射至 OpenGL 管线，为：
  - 颜色映射、
  - 等值线/等值面提取、
  - 动态粒子渲染
  等后续模块提供统一的数据接口。

### 1.3 对比分析目标

- 与 VTK / ParaView 等工具的对比维度：
  - 梯度计算速度（CPU vs GPU、串行 vs 并行）；
  - 数据降噪效果、多尺度融合质量（平滑程度 vs 特征保持）；
  - GPU 加速效率、资源占用；
- 当前代码中已通过 `vtkGradientFilter` 与自研 OpenGL 梯度引擎进行：
  - 时间开销对比（CPU wall time / GPU time）；
  - 数值误差评估（MAE、RMSE、MAXE）；
  - 梯度场可视化对比（梯度模长场）。

### 1.4 成果形式

- 一套 OpenGL 驱动的 CAE 数据预处理系统源码；
- 本科毕设论文与答辩材料。

## 2 系统总体设计

系统整体采用 **分层架构**，从底层到上层如下：

1. **OpenGL 上下文与运行时层**
   - 模块：`OpenGLManager` + `gl_context_utils`
   - 功能：通过 VTK 创建 OpenGL 上下文（支持离屏），使用 GLAD 加载函数指针，检查 OpenGL 版本（>= 4.3 支持计算着色器），记录 GPU 环境信息。

2. **GPU 计算核心层（梯度引擎）**
   - 模块：`GLGradientEngine` + `Shaders/FD.glsl`、`Shaders/WLS.glsl`
   - 功能：封装基于 SSBO 的计算管线，支持：
     - 规则网格的有限差分梯度计算；
     - 非结构化网格的加权最小二乘梯度计算；
     - GPU 计时查询（性能分析）。

3. **内部数据模型与转换层**
   - 模块：`DataObject`、`VTKDataConverter`
   - 功能：
     - 以内存友好的格式保存网格与场数据（包含 CSR 邻接信息）；
     - 在 `vtkDataSet` 与 `DataObject` 之间双向转换；
     - 支持规则网格与非结构化网格的统一表示。

4. **业务接口/门面层**
   - 模块：`CAEInterfaceTypes`、`CAEProcessingFacade`
   - 功能：
     - 对外提供统一的 API（dataset 装载、字段枚举、梯度计算、结果导出）；
     - 隐藏 OpenGL / VTK / 数据结构细节；
     - 对 Qt UI 或测试程序表现为“黑盒服务”。

5. **验证与示例层**
   - 模块：`TestGradient`、`TestConvert1`（测试代码）
   - 功能：
     - 验证 VTK 转换正确性（网格/邻接结构/数据数组）；
     - 对比 OpenGL 梯度引擎与 `vtkGradientFilter` 的结果与性能；
     - 输出可视化窗口（VTK 双视口，比较不同梯度结果）。

6. **Qt UI 层（设计中）**
   - 模块：Qt 主窗口、数据管理面板、参数面板、渲染窗口等（尚未实现）；
   - 功能：
     - 通过 `CAEProcessingFacade` 调用底层功能；
     - 提供数据导入/选择字段/配置算法参数/执行预处理/查看可视化结果的完整交互流程。

## 3 数据模型设计（DataObject）

### 3.1 内部数据结构

核心类型定义见 `DataObject.h`：

- `enum DataArrayType`
  - `POINT_DATA`：点数据
  - `CELL_DATA`：单元数据

- `struct DataArray`
  - `name`：数组名称；
  - `data`：`std::vector<float>`，以平铺方式存储；
  - `numComponents`：每个元组的分量数；
  - `dataType`：点/单元数据。

- `enum GridType`
  - `DATA_OBJECT_TYPE_RegularGrid`：规则网格；
  - `DATA_OBJECT_TYPE_UNSTRUCTURED`：非结构化网格。

- `class DataObject`：内部统一数据模型
  - 几何拓扑：
    - `std::vector<float> points`：点坐标 `[x0,y0,z0,x1,y1,z1,...]`；
    - `std::vector<float> cellCenters`：单元中心坐标；
    - `std::vector<int> cells`：单元连接表；
    - `std::vector<int> cellTypes`：VTK 单元类型；
    - `std::vector<int> cellOffsets`：CSR 风格的单元起始索引；
  - 邻接关系（CSR 形式）：
    - `pointNeighbors` / `pointNeighborOffsets`：点–点邻居；
    - `pointInCellNeighbors` / `pointInCellNeighborOffsets`：点–单元关联；
    - `cellNeighbors` / `cellNeighborsOffsets`：单元–单元邻接；
  - 数据数组：
    - `std::vector<DataArray> dataArrays`：所有场数据（标量/向量/张量，点或单元）；
  - 网格维度：
    - `int dimensions[3]`：规则网格的 (nx, ny, nz)；
  - 常用查询：
    - `findDataArray`：按名称和点/单元类型查找；
    - `upsertDataArray`：插入或更新数组；
    - `pointCount` / `cellCount`：便捷计数接口。

### 3.2 邻接信息与梯度计算

- 对 **规则网格**：
  - 通过 `dimensions[3]` 与 `points` 构建隐式拓扑；
  - 在梯度计算中可按 (i, j, k) 索引邻域，不需要显式邻接表。
- 对 **非结构化网格**：
  - 通过 `VTKDataConverter` 中的拓扑遍历和/或 KNN 构建 `pointNeighbors` / `cellNeighbors`；
  - 为 **WLS 梯度** 和后续 **滤波/平滑** 提供局部邻域支持。

内存布局采用线性数组 + CSR 的形式，兼顾了：

- 适合 GPU 端以 SSBO 形式读取（顺序访问）；
- 适合 CPU 端快速遍历与缓存局部性。

## 4 VTK 转换设计（VTKDataConverter）

### 4.1 VTK -> 内部数据

`VTKDataConverter` 主要方法设计：

- `bindVTKDataAndInternalData(vtkDataSet*, DataObject*)`
  - 将一个 VTK 数据集和一个内部 `DataObject` 绑定，为后续转换做好准备。

- `convertVTKToInternal()`
  - 总控入口：
    - 先通过 `convertType()` 判定网格类型（规则 / 非结构化）；
    - 规则网格调用 `convertRegularGrid()`；
    - 非结构化网格调用 `convertUnstructuredGrid()`。

- 规则网格转换 `convertRegularGrid()`：
  - `convertPoints()`：拷贝点坐标；
  - `convertDimensions()`：填充 `dimensions`；
  - `convertDataArrays()`：点/单元所有数据数组；
  - `convertCellCenters()`：计算单元中心；
  - `convertCell()`：填充单元连接与类型。

- 非结构化网格转换 `convertUnstructuredGrid()`：
  - `convertPoints()`；
  - `convertDataArrays()`；
  - `convertPointInCellNeighbors()`：基于 `vtkStaticCellLinks` 记录点所在单元；
  - `convertPointNeighborsRobust(minK=4, knnK=24)`：
    - 先用拓扑（共单元）构建邻接；
    - 若邻居不足 `minK`，使用 KNN 补充；
  - `convertCellCenters()`；
  - `convertCell()`；
  - `convertCellNeighborsByKNN(k=5)`：以单元中心为点，构建 KNN 邻接。

通过上述流程，内部 `DataObject` 获得：

- 准确的几何与拓扑结构；
- 完整的点/单元数据数组；
- 稳定的邻接关系（兼顾稀疏区域）。

### 4.2 内部数据 -> VTK

`convertInternalToVTK()` 支持：

- 根据 `gridType` 生成 `vtkStructuredGrid` 或 `vtkUnstructuredGrid`；
- 使用 `points` 填充 `vtkPoints`；
- 使用 `cells` / `cellOffsets` / `cellTypes` 恢复单元拓扑；
- 遍历 `dataArrays`：
  - 若为 `POINT_DATA` 且 tuple 数与点数一致，则写入 `PointData`；
  - 若为 `CELL_DATA` 且 tuple 数与单元数一致，则写入 `CellData`。

该双向转换支撑两类使用场景：

1. **验证**：将 GPU 计算出的梯度场（存放在 `DataObject` 中）再转回 VTK，借助 VTK/ParaView 进行可视化；
2. **集成**：将预处理结果提供给其他只接受 VTK 数据的可视化或后处理模块。

## 5 梯度计算与 GPU 计算引擎设计（GLGradientEngine）

### 5.1 Compute Shader 设计思路

- 环境要求：OpenGL 4.3+，支持 Compute Shader；
- 数据传输：使用 SSBO（Shader Storage Buffer Object）存放：
  - 点或单元中心位置；
  - 输入场数据（标量/向量/张量）；
  - 输出梯度结果；
  - 非结构化网格的邻接结构（offset + neighbors）。
- 调度：
  - 规则网格：按三维工作组 (gx, gy, gz) 启动；
  - 非结构化网格：按点/单元数线性划分，使用 1D 工作组。

### 5.2 FD 梯度（规则网格）

`computeRegularFD` 接口：

- 输入：
  - `positions`：规则网格上点或单元中心坐标（按 `dims` 排列）；
  - `values`：场值数据，支持多分量；
  - `RegularParams`：
    - `dims[3]`：三维网格尺寸；
    - `origin[3]`、`spacing[3]`：当前实现中未在 Compute Shader 中显式使用，可为扩展保留。
- 输出：
  - `outGrad`：大小为 `N * 3 * numComponents` 的梯度数组。

算法本质是多分量扩展版的三维有限差分。边界条件由着色器定义（例如前向/后向差分或简单复制）。

### 5.3 WLS 梯度（非结构化网格）

`computeUnstructuredWLS` 接口：

- 输入：
  - `positions`：点坐标或单元中心坐标；
  - `offsets` / `neighbors`：CSR 形式的邻接表；
  - `phi`：场值，支持多分量；
  - `WLSParams`：权重指数 `wExponent` 与正则化参数 `lambda`。
- 输出：
  - `outGrad`：大小为 `N * 3 * numComponents` 的梯度数组。

每个线程针对一个点（或单元）执行局部最小二乘拟合：

- 根据邻居相对位置构建局部矩阵；
- 使用距离权重（如 `exp(-dist^wExponent)`）；
- 通过正则化求解以增强数值稳定性。

### 5.4 性能计时与分析

- `setEnableGpuTiming(true)` 后，梯度计算前后会插入 `GL_TIME_ELAPSED` 查询；
- 查询结果（纳秒）转换为毫秒，保存在 `lastGpuTimeMs` 中；
- 上层通过 `CAEProcessingFacade::getLastComputeGpuMs()` 获取。

## 6 统一业务接口设计（CAEProcessingFacade）

### 6.1 接口 DTO（CAEInterfaceTypes）

- `CAEFieldAssociation`：`Point` / `Cell`
- `CAEGridClass`：`Regular` / `Unstructured`
- `CAEGradientMethod`：`Auto` / `FiniteDifference` / `WeightedLeastSquares`

- `struct CAEFieldInfo`
  - 字段名称、关联（点/单元）、分量数、tuple 数量。

- `struct CAEGradientRequest`
  - `datasetId`：数据集标识；
  - `inputArrayName`：输入场名；
  - `association`：点/单元；
  - `method`：梯度方法（Auto/FD/WLS）；
  - `wlsExponent` / `wlsLambda`：WLS 参数。

- `struct CAEGradientResultMeta`
  - 输出数组名、来源数组名、关联、方法、输入/输出分量数；
  - 计算 wall time 与 GPU time。

- `struct CAEDatasetSummary`
  - 数据集 id / 名称；
  - 网格类别；
  - 点/单元数量；
  - 所有场信息列表；
  - 所有梯度结果列表。

以上 DTO 由 UI / 测试程序使用，屏蔽内核细节。

### 6.2 Facade 行为设计

`CAEProcessingFacade` 对外提供的主要功能：

1. **初始化**
   - `initialize(shaderDir)`
     - 调用 `OpenGLManager` 创建 OpenGL 上下文；
     - 设置着色器目录（如 `"Shaders"`），初始化 `GLGradientEngine`；
     - 打开 GPU 时间记录。

2. **数据集管理**
   - `loadDatasetFromVTKFile(filePath)`
     - 使用 `vtkDataSetReader` 读取 VTK Legacy 文件；
     - 通过 `VTKDataConverter` 转换为内部 `DataObject`；
     - 分配 `datasetId`，存入 `m_records`；
     - 返回该 id。
   - `listDatasets()`
     - 返回当前所有加载数据集的 `CAEDatasetSummary` 列表。
   - `getDatasetSummary(datasetId, outSummary)`
   - `listFields(datasetId, assoc, outFields)`
     - 筛选指定点/单元关联下可用于计算的字段列表。

3. **梯度计算**
   - `computeGradient(const CAEGradientRequest& req, CAEGradientResultMeta& outMeta)`
     - 根据 `datasetId` 和 `inputArrayName` 获取 `DataArray`；
     - 若 `method == Auto`：
       - 规则网格 => `FiniteDifference`；
       - 非结构化网格 => `WeightedLeastSquares`；
     - 根据最终方法调用内部：
       - `computeByFD`（规则网格）；
       - `computeByWLS`（非结构化，点或单元）。
     - 将结果存回对应 `DataObject` 的 `dataArrays`（使用 `upsertDataArray`）；
     - 记录 `CAEGradientResultMeta`，并返回。
   - 性能信息：
     - `getLastComputeWallMs()`：CPU 侧整体耗时；
     - `getLastComputeGpuMs()`：GPU 端计算耗时。

4. **结果访问与导出**
   - `getArrayData(datasetId, arrayName, assoc, outData, outComps)`
     - 获取指定数组的数据内容和分量数；
   - `exportDatasetToVTK(datasetId, outVtk)`
     - 通过 `convertInternalToVTK` 将内部数据集导出为 `vtkDataSet`，用于渲染或进一步处理。

### 6.3 Qt UI 预期集成方式（设计）

Qt UI 层可按以下流程使用 Facade：

1. “打开文件”按钮：
   - 调用 `loadDatasetFromVTKFile`；
   - 刷新数据集列表。

2. 字段选择：
   - 使用 `listDatasets` + `getDatasetSummary` 或 `listFields` 填充 ComboBox。

3. 梯度计算：
   - 根据用户选择构造 `CAEGradientRequest`（方法、点/单元、WLS 参数等）；
   - 调用 `computeGradient`；
   - 根据返回的 `CAEGradientResultMeta` 更新字段列表。

4. 可视化渲染：
   - 使用 `exportDatasetToVTK` 获得 `vtkDataSet`；
   - 在 Qt 中嵌入 VTK 渲染窗口（或自定义 OpenGL 渲染）；
   - 选择合适的标量场进行颜色映射/等值面等。

后续数据降噪与多尺度接口设计可以复用同样模式，例如增加：

- `computeSmoothedField(...)`；
- `computeMultiScaleFusion(...)` 等在 Facade 中的高层 API。

## 7 数据优化、降噪与多尺度融合设计（规划）

目前代码重点完成了 **梯度计算相关模块**，数据降噪与多尺度部分仍处于设计/规划阶段。预期设计如下：

### 7.1 数据降噪（双边滤波）

- 算法思路：
  - 在点或单元级别进行 **空间 + 数值域** 的联合滤波；
  - 邻域范围由 `pointNeighbors` / `cellNeighbors` 决定；
  - 核函数结合空间距离和物理量差异，避免过度平滑梯度高区域。

- 实现方式：
  - 优先考虑 **OpenGL Compute Shader** 实现，复用 SSBO 邻接结构；
  - 对规则网格可采用体素双边滤波，与 FD 梯度相似的网格访问模式。

### 7.2 多尺度数据构建

- 思路：
  - 通过迭代应用双边滤波，构建多个平滑尺度（如 coarse / medium / fine）；
  - 或采用金字塔结构（down-sampling + up-sampling）构造层级。

- 存储：
  - 以多个 `DataArray` 的形式保存在同一 `DataObject` 中，如：
    - `field_s1`, `field_s2`, `field_s3` 等。

### 7.3 特征保持的多尺度融合

- 目标：
  - 在平滑噪声的同时保留重要特征（高梯度区域，例如边界层、裂纹等）。

- 思路（示例）：
  - 在不同尺度上比较梯度模长；
  - 对梯度大的区域给予细尺度更高权重；
  - 对平滑区域采用粗尺度数据；
  - 最终融合为一个平衡噪声与特征的结果场。

- 接口设计建议：
  - 在 Facade 增加高层接口，如：
    - `computeMultiScaleSmoothedField(const MultiScaleRequest&, MultiScaleResultMeta&)`；
  - 复用现有 `CAEFieldInfo` / `DataArray` 机制，无需改变底层数据结构。

### 7.4 异常值检测（规划）

- 基于局部统计（邻域均值/方差）或分位数检测异常值；
- 结果可输出：
  - 一个布尔掩码数组；
  - 或自动在滤波/融合过程中降低异常样本权重。

## 8 对比验证与实验设计

当前 `TestGradient` 中已经实现了 **OpenGL 梯度引擎 vs VTK 梯度滤波** 的基准测试流程：

1. 从 VTK 文件读取数据集；
2. 使用 `CAEProcessingFacade` 在 GPU 侧计算梯度；
3. 使用 `vtkGradientFilter` 在 CPU 侧计算梯度；
4. 比较：
   - 运行时间：平均值 + 最小值（多次重复）；
   - 数值误差：MAE / RMSE / 最大误差；
5. 将结果写回 VTK 数据集，比较梯度模长场的可视化效果。

未来实验设计可扩展为：

- 对多组数据集进行统计；
- 增加双边滤波、多尺度融合前后的效果对比；
- 与 ParaView 中相同滤波器（若有类似功能）进行对照。

  - `DataObject`：统一数据模型设计完备，已支持点、单元、邻接、维度等信息；
  - `VTKDataConverter`：
    - 支持规则/非结构化网格；
    - 支持邻接信息构建（包括 KNN 与拓扑混合策略）；
    - 支持 `DataObject` ↔ `vtkDataSet` 双向转换；
  - `TestConvert1`（注释代码）曾用于验证转换正确性，目前仍可参考。

- GPU 梯度引擎：
  - `GLGradientEngine`：
    - 支持基于 Compute Shader 的 FD / WLS 梯度计算；
    - 支持多分量场；
    - 支持 GPU 时间测量；
  - Shaders：
    - `Shaders/FD.glsl` 与 `Shaders/WLS.glsl` 已用于实际计算。

- 统一接口与测试：
  - `CAEInterfaceTypes`：定义了对外 API 需要的数据结构；
  - `CAEProcessingFacade`：
    - 集成 VTK 转换 + OpenGL 梯度引擎；
    - 实现 `initialize` / `loadDatasetFromVTKFile` / `computeGradient` / `exportDatasetToVTK` 等关键接口；
  - `TestGradient`：
    - 使用 Facade 进行梯度计算；
    - 与 `vtkGradientFilter` 做性能和精度对比；
    - 提供简单 VTK 渲染窗口进行结果可视化。

- 初步接口文档：
  - `doc/接口文档.md`：已有一份概要性的接口说明。

### 9.2 尚未完成或规划中的部分

- 数据降噪、特征保持平滑、多尺度融合：
  - 目前尚未在 C++/GL 中落地具体实现；
  - Facade 中还未添加对应 API；
  - Shader 层也未添加相应计算着色器。

- 异常值检测：
  - 未实现；
  - 需要设计合适的指标（局部统计、梯度异常等）。

- Qt 图形界面：
  - 当前项目仍主要通过命令行测试程序运行；
  - Qt UI（文件管理、字段选择、参数调节、渲染视图等）尚未实现；
  - 与 Facade 的交互逻辑已基本清晰，待代码实现。

- 更系统的对比实验：
  - 目前仅实现了与 `vtkGradientFilter` 的时间/误差对比；
  - 尚未对降噪、多尺度融合等方面与 ParaView 等进行量化对比。
