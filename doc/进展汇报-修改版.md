# 进展汇报（修改版）

针对上次汇报中老师提出的两个核心问题，本阶段我主要完成了以下调整。

第一，针对非结构化网格梯度计算中原加权最小二乘法与 VTK 结果偏差较大的问题，我已将当前主方法调整为与 `vtkGradientFilter` 更一致的基于形函数导数的梯度计算方法。当前系统支持一阶三角形、四边形、四面体和六面体四种非结构化单元类型的梯度计算。

第二，针对数据优化模块中“实验扰动与真实 CAE 数据扰动原因不够对应”的问题，我准备将模块表述收缩为：面向一类局部随机高频数值扰动的抑制与边缘保持处理，从而使研究对象、实验方式和结论边界保持一致。

## 1. 梯度计算模块

### 1.1 方法调整

此前非结构化网格部分采用的是基于加权最小二乘法的梯度计算方法。该方法可以作为一种工程近似方法，但它与 `vtkGradientFilter` 的数学路线并不一致，因此在直接将系统结果与 VTK 结果进行对比时，会同时混入“算法原理不同”和“实现误差不同”两类因素，导致对比不够严谨。

基于这一问题，当前我已将非结构化网格的主方法调整为参考 `vtkGradientFilter` 的基于形函数导数的梯度计算方法。这样做的目的不是简单追求与 VTK 数值接近，而是先保证算法路线一致，再在此基础上讨论实现正确性、工程一致性和 GPU 计算效率。

当前系统状态如下：

- 规则网格继续采用有限差分法。
- 非结构化网格默认采用基于形函数导数的方法。
- 梯度计算在 OpenGL 侧完成，作为当前系统主实现。
- 与 VTK 的结果对比和时间对比都以这条方法路线为准。

当前支持的非结构化单元类型如下：

| 单元类型 | 阶次 | 节点数 | 当前状态 |
| --- | --- | ---: | --- |
| Triangle | 一阶 | 3 | 已支持 |
| Quad | 一阶 | 4 | 已支持 |
| Tetra | 一阶 | 4 | 已支持 |
| Hexahedron | 一阶 | 8 | 已支持 |

也就是说，当前论文中我准备将非结构化网格梯度计算功能明确表述为：支持一阶三角形、四边形、四面体和六面体单元的基于形函数导数的梯度计算。

### 1.2 实验组织

围绕梯度计算模块，当前实验内容分为三类。

1. 解析场验证实验。该实验用于验证算法本身的正确性，即将系统计算结果与解析真值进行对比。当前选取的数据集为 `SampleStructGrid`、`hexa` 和 `1_0`，分别对应规则网格、非结构化三维网格和非结构化二维曲面网格。
2. 与 VTK 结果一致性对比实验。该实验用于验证在真实数据字段上，当前系统与 `vtkGradientFilter` 的工程输出是否一致。该部分分别对点数据和单元数据进行测试。
3. 与 VTK 的时间对比实验。该实验用于在控制变量成立的前提下，对比系统实现与 VTK 并行实现的计算时间，分析当前 OpenGL 实现的工程效率。该部分不再直接采用异构真实模型，而是采用统一生成的同构网格族进行测试，以保证实验结论具有可比性和可解释性。

### 1.3 解析场验证实验

解析场验证实验的目标是验证算法本身的正确性，因此该部分只比较系统结果与解析真值之间的误差，不直接与 VTK 结果进行比较。

其中，`SampleStructGrid` 和 `hexa` 采用三维解析场；`1_0` 作为曲面网格，采用曲面切向解析场，并以曲面内在梯度指标作为主口径，从而避免法向分量对结论的干扰。

#### 1.3.1 线性标量场

线性标量场实验中，三维数据集在归一化局部坐标 `(x_r, y_r, z_r)` 上构造一次标量函数；`1_0` 数据集在曲面局部切向坐标 `(u, v)` 上构造一次标量函数。实验结果如下。

| 数据集 | 场函数说明 | NRMSE |
| --- | --- | ---: |
| `SampleStructGrid` | 三维线性标量场 `benchmark_linear` | `2.82038e-07` |
| `hexa` | 三维线性标量场 `benchmark_linear` | `1.57998e-07` |
| `1_0` | 曲面切向线性标量场 `benchmark_surface_linear` | `1.68088e-07` |

#### 1.3.2 线性向量场

线性向量场实验中，三维数据集在归一化局部坐标上分别为三个分量构造一次函数；`1_0` 数据集在曲面局部切向坐标上分别构造各分量，并投影到曲面局部支撑空间。实验结果如下。

| 数据集 | 场函数说明 | NRMSE |
| --- | --- | ---: |
| `SampleStructGrid` | 三维线性向量场 `benchmark_vec_linear` | `3.30256e-07` |
| `hexa` | 三维线性向量场 `benchmark_vec_linear` | `2.36149e-07` |
| `1_0` | 曲面切向线性向量场 `benchmark_surface_vec_linear` | `1.32859e-07` |

从以上结果可以看出，当前系统在规则网格、三维六面体非结构化网格以及二维曲面四边形网格上的点数据解析场验证结果均达到 `1e-7` 量级，说明当前梯度计算实现能够正确恢复线性标量场和线性向量场的梯度信息，可作为算法正确性的直接证据。

### 1.4 与 VTK 结果一致性对比实验

与 VTK 结果一致性对比实验的目标是验证：在真实数据字段上，当前系统与 `vtkGradientFilter` 的工程输出是否一致。该实验并不单独证明算法的数学正确性，而是用于说明当前实现与 VTK 在工程结果上的一致性。

#### 1.4.1 点数据

| 数据集 | 字段 | NRMSE |
| --- | --- | ---: |
| `SampleStructGrid` | `scalars` | `8.22437e-08` |
| `hexa` | `scalars` | `6.27285e-08` |
| `1_0` | `RF` | `5.63477e-08` |

#### 1.4.2 单元数据

| 数据集 | 字段 | NRMSE |
| --- | --- | ---: |
| `SampleStructGrid` | `scalars` | `7.62786e-07` |
| `limb` | `chem_0` | `3.89516e-07` |
| `1_0` | `S_Mises` | `7.77806e-08` |

实验结果表明，在当前展示的规则网格、二维曲面四边形网格和三维六面体网格案例上，系统结果与 `vtkGradientFilter` 保持了较高一致性，主要误差指标处于 `1e-7` 到 `1e-6` 量级，可说明当前实现具有较好的工程一致性。

### 1.5 与 VTK 的时间对比实验

与 VTK 的时间对比实验用于分析当前 OpenGL 实现的工程效率。为保证实验严谨性，该部分不直接采用异构真实模型，而是构造统一几何域、统一输入场和统一字段名的同构网格族进行测试。

当前采用两组网格族：规则网格族 `timing_struct_*` 与代表性非结构化六面体网格族 `timing_uhex_*`。实验中统一采用 `POINT` 路径，统一字段名为 `scalars`，VTK 并行 backend 为 `STDThread`，并固定 VTK 并行线程数为 `16`。

#### 1.5.1 规则网格

| 数据集 | 点数/单元数 | VTK 并行线程数 | VTK 并行时间/ms | 系统总时间/ms | GPU 时间/ms |
| --- | --- | ---: | ---: | ---: | ---: |
| `timing_struct_20x20x20` | `8000 / 6859` | `16` | `2.7573` | `0.9036` | `0.025272` |
| `timing_struct_32x32x32` | `32768 / 29791` | `16` | `3.5564` | `1.4336` | `0.043732` |
| `timing_struct_48x48x48` | `110592 / 103823` | `16` | `8.2716` | `4.0996` | `0.107692` |

#### 1.5.2 非结构化网格

| 数据集 | 点数/单元数 | VTK 并行线程数 | VTK 并行时间/ms | 系统总时间/ms | GPU 时间/ms |
| --- | --- | ---: | ---: | ---: | ---: |
| `timing_uhex_20x20x20` | `8000 / 6859` | `16` | `163.799` | `2.8527` | `0.623896` |
| `timing_uhex_32x32x32` | `32768 / 29791` | `16` | `708.650` | `5.3835` | `2.53781` |
| `timing_uhex_48x48x48` | `110592 / 103823` | `16` | `3349.36` | `44.8039` | `7.91991` |

从时间实验结果可以看出，在当前测试条件下，系统的系统总时间与 GPU 时间均明显低于 VTK 并行时间，且随数据规模增大呈现稳定增长趋势。这说明当前基于 OpenGL 的梯度计算实现具备较稳定的 GPU 计算性能。

需要说明的是，系统总时间反映的是算法调用级总耗时，GPU 时间反映的是 OpenGL 侧纯计算时间，而 VTK 并行时间对应的是 `vtkGradientFilter::Update()` 的执行时间，因此三者在统计口径上并不完全相同。本文将系统总时间与 VTK 并行时间作为主对比口径，将 GPU 时间作为补充指标，用于说明 OpenGL 侧的纯计算代价。

## 2. 数据优化模块

### 2.1 表述调整

对于之前数据优化模块实验中“人为添加噪声与真实 CAE 数据扰动原因不够对应”的问题，我准备将模块表述调整如下。

CAE 仿真数据会因离散化误差、迭代收敛误差与舍入误差等因素产生局部数值扰动。对于其中一类表现为局部随机高频波动的扰动，可采用高斯扰动作为代理模型进行近似描述。因此，本模块面向这类局部数值扰动，实验上通过在干净场上叠加高斯扰动，并比较优化前后的误差与粗糙度指标，验证模块对局部随机高频扰动的抑制能力及其边缘保持特性。

### 2.2 实验结果

当前实验在 `ShipHull_0` 数据集上构造多类干净场，并分别添加高斯扰动。下面给出点数据与单元数据路径上的平均结果。

| 关联方式 | 输入 NRMSE | 输出 NRMSE | RMSE 改进比 | 粗糙度比 |
| --- | ---: | ---: | ---: | ---: |
| `POINT` | `0.296642` | `0.180858` | `0.610772` | `0.406530` |
| `CELL` | `0.294558` | `0.121921` | `0.414806` | `0.387052` |

从实验结果可以看出，无论是点数据还是单元数据，经过数据优化后，输出误差均低于输入误差，说明该模块对局部随机高频扰动具有稳定的抑制效果。进一步结合粗糙度指标进行分析，还可以说明该模块在抑制局部波动的同时，能够较好地保持场中主要结构特征。
