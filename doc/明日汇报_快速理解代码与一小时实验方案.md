## 5. 20 分钟速读路线

## 5.1 第 1 步：先看 `CAEProcessingFacade.h`，只看它对外提供了什么

你先不要看实现，先看接口。

你只要记住 `CAEProcessingFacade` 负责 5 件事：

1. `initialize(...)`
   - 初始化 OpenGL 和 shader。
2. `loadDatasetFromVTKFile(...)`
   - 读取数据并转换成内部结构。
3. `computeGradient(...)`
   - 算梯度。
4. `computeMultiScaleDecompositionAndFusion(...)`
   - 做数据优化。
5. `saveDatasetToVTKFile(...)`
   - 导出结果给 ParaView。

这一步最重要，因为你明天汇报时，整个系统都可以围绕这个类来讲。

---

## 5.2 第 2 步：看 `CAEInterfaceTypes.h`，弄清楚请求和结果长什么样

这一层你只需要记住 4 个结构体：

1. `CAEGradientRequest`
   - 梯度计算需要什么参数。
2. `CAEGradientResultMeta`
   - 梯度计算会返回什么元信息。
3. `CAEMultiScaleRequest`
   - 多尺度优化需要什么参数。
4. `CAEMultiScaleResultMeta`
   - 多尺度优化生成哪些结果数组。

这一层的作用是：

1. GUI、测试程序、门面层之间，用统一请求对象沟通。
2. 你明天讲“系统不是散着写的，而是有统一接口层”，就靠这里。

---

## 5.3 第 3 步：看 `DataObject.h`，弄清楚内部数据怎么存

这里只记 6 个核心成员：

1. `points`
   - 点坐标。
2. `cellCenters`
   - 单元中心坐标。
3. `dataArrays`
   - 点/单元字段数组。
4. `pointNeighbors + pointNeighborOffsets`
   - 点邻域图。
5. `cellNeighbors + cellNeighborsOffsets`
   - 单元邻域图。
6. `dimensions`
   - 规则网格尺寸。

你可以这样理解：

> `DataObject` 就是项目内部专门为 GPU 计算准备的统一数据表示。

---

## 5.4 第 4 步：看 `VTKDataConverter.h`，弄清楚 VTK 是怎么变成内部数据的

这里只需要记住两个总入口：

1. `convertVTKToInternal()`
2. `convertInternalToVTK()`

然后记住：

1. 规则网格和非结构网格会走不同转换流程；
2. 非结构网格会额外构建点邻域和单元邻域；
3. 这些邻域后面要给 AWLS 和图滤波用。

这一步是你明天解释“为什么系统可以统一处理不同网格”的关键。

## 5.6 第 6 步：看 `GLGradientEngine.h` 和 `GLFilterEngine.h`

今晚不要追着实现细节看，只看“这个引擎负责什么”。

### `GLGradientEngine`

只记住 3 个函数：

1. `computeRegularFD(...)`
2. `computeUnstructuredWLS(...)`
3. `computeUnstructuredAdaptiveWLS(...)`

意思是：

1. 规则网格走 FD；
2. 非结构网格主线走 AWLS；
3. 传统 WLS 主要作为回退或对照。

### `GLFilterEngine`

只记住 2 个函数：

1. `bilateralGraph(...)`
2. `fuseMultiScale(...)`

意思是：

1. 先图双边滤波平滑；
2. 再做多尺度细节融合。

## 6. 今晚你必须记住的 3 条代码主线

## 6.1 主线一：数据是怎么走的

```text
.vtk 文件
-> vtkDataSet
-> VTKDataConverter
-> DataObject
-> CAEProcessingFacade
-> GLGradientEngine / GLFilterEngine
-> 结果写回 DataObject
-> 测试程序 / GUI / VTK 导出
```

## 6.2 主线二：梯度模块怎么分派

```text
如果是规则网格 -> FD
如果是非结构网格 -> AWLS
```

## 6.3 主线三：数据优化模块怎么做

```text
输入字段
-> 多层图双边滤波
-> 相邻层做差得到 detail
-> base + detail 融合得到 fused
```
